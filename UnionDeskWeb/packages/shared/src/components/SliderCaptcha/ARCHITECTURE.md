## 核心算法

### 1. 轨迹验证算法

```typescript
function verifyTrack(track: TrackPoint[], threshold: number): VerifyResult {
  // 步骤 1: 检查轨迹点数量
  if (track.length < 2) {
    return { success: false, message: '滑动轨迹过短' };
  }

  // 步骤 2: 计算总时长
  const duration = track[track.length - 1].t - track[0].t;
  if (duration < 100) {
    return { success: false, message: '滑动速度过快' };
  }

  // 步骤 3: 计算平均速度
  const distance = track[track.length - 1].x - track[0].x;
  const avgSpeed = distance / duration;
  if (avgSpeed > 5) {
    return { success: false, message: '滑动速度异常' };
  }

  // 步骤 4: 检查滑动距离
  const finalPosition = track[track.length - 1].x;
  if (finalPosition < threshold * 0.95) {
    return { success: false, message: '未滑动到最右边' };
  }

  // 步骤 5: 生成 token
  return {
    success: true,
    token: generateToken(),
  };
}
```

### 2. Token 生成算法

```typescript
function generateToken(): string {
  // 格式: {timestamp}_{random16}
  // 示例: 1745366400000_aB3dE5fG7hI9jK1L
  
  const timestamp = Date.now();
  const random = generateRandomString(16);
  return `${timestamp}_${random}`;
}
```

```typescript
const handleMove = useCallback((clientX: number) => {
  if (!isDraggingRef.current) return;
  
  const moveX = clientX - startX;
  const newLeft = Math.min(Math.max(moveX, 0), maxDistance);
  
  setSliderLeft(newLeft);
  trackRef.current.push({ x: newLeft, t: Date.now() });
}, [startX, maxDistance]);
```

## 总结

SliderCaptcha 组件采用了清晰的分层架构，将 UI、事件处理、业务逻辑和类型定义分离，确保了代码的可维护性和可扩展性。通过轨迹分析和速度检测实现了基础的安全防护，同时保持了极简的用户体验。组件设计遵循了 React 最佳实践，使用 Hooks 进行状态管理，使用 TypeScript 提供类型安全，是一个生产级别的验证组件实现。
