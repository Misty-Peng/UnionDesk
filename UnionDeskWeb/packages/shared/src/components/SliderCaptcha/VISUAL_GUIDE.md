# SliderCaptcha 视觉效果指南

## 组件状态展示

### 1. 初始状态（idle）

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│  [>>]  ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  │
│        请按住滑块，拖动到最右边                             │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

**特征：**
- 背景：浅灰色 (#f5f5f5)
- 滑块：白色，位于最左侧
- 未滑动区域：斜纹纹理（45度）
- 提示文字：灰色 (#999)
- 边框：1px 灰色 (#d9d9d9)

---

### 2. 拖动中（moving）

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│  ████████████[>>]░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  │

│                                                            │
└────────────────────────────────────────────────────────────┘
```

**特征：**
- 背景：保持浅灰色
- 滑过区域：浅蓝色 (#1890ff, 透明度 15%)
- 边界线：2px 蓝色实线，标识滑过区域边界
- 未滑动区域：斜纹纹理
- 滑块：跟随鼠标/手指移动
- 提示文字：灰色

---

```

```

│                                                            │

**特征：**
- 背景：红色 (#ff4d4f)
- 滑过区域：红色（透明度 15%）
- 滑块：位于失败位置，图标变为红色叉号
- 提示文字：白色
- 状态：1秒后自动重置为初始状态

- 恢复初始状态
- 可以重新开始拖动

| 状态 | 背景色 | 进度条颜色 | 文字颜色 |
|------|--------|-----------|---------|
| idle | #f5f5f5 | - | #999 |
| moving | #f5f5f5 | #1890ff (15%) | #999 |

|------|------|------|

## 响应式设计

### PC 端
- 鼠标悬停：滑块显示 `cursor: grab`
- 拖动时：滑块显示 `cursor: grabbing`
- 边界线：2px 实线

### 移动端
- 触摸反馈：滑块轻微放大（可选）
- 边界线：2px 实线
- 触摸区域：滑块周围增加 8px 触摸区域（可选）

---

## 无障碍设计

### 视觉对比度
- 文字与背景对比度：≥ 4.5:1（WCAG AA 标准）
- 边界线与背景对比度：≥ 3:1

### 状态指示
- 成功：绿色背景 + 对勾图标
- 失败：红色背景 + 叉号图标
- 不依赖颜色单独传达信息

## 总结

SliderCaptcha 组件的视觉设计遵循以下原则：

1. **清晰的状态区分**：通过颜色、纹理、边界线明确区分不同状态
2. **流畅的动画效果**：0.3秒过渡动画，提供平滑的视觉反馈
3. **统一的设计语言**：与 Ant Design 风格保持一致
4. **良好的可访问性**：高对比度，不依赖颜色单独传达信息
5. **响应式设计**：同时支持 PC 和移动端

通过这些设计，组件既美观又实用，为用户提供了优秀的交互体验。
