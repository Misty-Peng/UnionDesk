# UnionDesk 权限矩阵

> 本文档完整列出所有角色与权限项的对应关系，供前后端权限校验、角色配置和 QA 验证使用。
> 来源：`产品需求文档 v1.0.md § 2.1`

---

## 1. 角色体系

### 1.1 平台角色（全局）

| 角色 | 代码 | 预设 | 说明 |
|------|------|------|------|
| 平台管理员 | `platform_admin` | 是 | 平台级最高权限，负责业务域创建、平台配置、全局授权与跨域治理 |

- 平台角色权限集内置不可编辑。
- `platform_admin` 权限高于任一业务域内的 `super_admin`；创建业务域时自动成为当前域 `super_admin`，手动授予/回收域级 `super_admin` 需二次验证并写入审计日志。
- 平台必须始终保留至少 1 个激活的 `platform_admin`；最后一个 `platform_admin` 不允许被禁用、删除、离职释放、降级或回收平台角色。

### 1.2 业务域角色（域内）

| 角色 | 代码 | 预设 | 说明 |
|------|------|------|------|
| 域超级管理员 | `super_admin` | 是，不可删除 | 当前业务域最高权限，可管理本域配置、成员、角色、权限与域级审计 |
| 域管理员 | `domain_admin` | 是，不可删除 | 管理本域常规配置、用户、角色、权限，不可授予或回收 `super_admin` |
| 客服 | `agent` | 是，不可删除 | 处理工单、接待咨询、使用知识库 |
| 自定义角色 | `custom_*` | 否 | 域管理员创建，最多 20 个/域 |

- 预设角色核心权限不可更改，可复制后修改为新角色。
- 用户最终权限 = 域内所有角色权限项并集。

### 3.3 数据范围

| 范围代码 | 说明 | `super_admin` 默认 | `domain_admin` 默认 | `agent` 默认 |
|----------|------|:---:|:---:|:---:|
| `self` | 仅自己 | — | — | ✅ |
| `domain_all` | 本域全部 | ✅ | ✅ | — |

- 数据范围通过权限项后缀表达（如 `ticket:view:self` / `ticket:view:domain_all`）。
- 角色编辑页不提供独立的数据范围配置面板。

## 4. 自定义角色规则

| 约束 | 规则 |
|------|------|
| 最大数量 | 每域最多 20 个自定义角色 |
| 权限项来源 | 仅可组合 §3.2 表中系统预置权限项 |
| 权限项新增 | 域管理员不可自行新增权限项代码 |
| 角色变更方式 | 单用户逐一调整，不支持批量变更 |
| 权限叠加 | 用户最终权限 = 所有角色权限项并集 |
| 预设角色 | 不可删除、核心权限不可修改、可复制 |

## 5. 账号自助能力（不进入域权限项）

以下操作由账号所有权和当前认证状态控制，不在域内权限项中体现：

| 操作 | 控制方式 |
|------|----------|
| 找回密码 | 短信优先→邮箱降级→管理员重置 |
| 修改密码 | 已登录状态 |
| 修改资料 | 已登录状态 |
| 改绑手机号/邮箱 | 二次验证 |
| 注销申请 | 已登录状态（仅客户） |

## 6. 敏感操作管控

| 操作 | 要求 |
|------|------|
| 手动授予/回收域级 `super_admin` | `platform_admin` 执行，二次验证 + 审计日志；创建业务域时自动成为当前域 `super_admin` |
| 删除业务域 | 每次需重新验证密码 |
| 身份归并 | 每次需重新验证密码 |
| 彻底删除员工/客户 | 每次需重新验证密码 |
| 全量导出 | 每次需重新验证密码 |
| 全局安全策略修改 | 每次需重新验证密码 |
| 其他敏感操作 | 验证后 15 分钟内同类操作放行 |
