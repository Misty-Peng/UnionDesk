# UnionDesk 状态机表

> 本文档定义系统内所有核心实体的状态流转规则，供前后端开发、QA 验证与数据校验使用。
> 来源：`产品需求文档 v1.0.md`

---

## 1. 工单状态机 (`ticket.status`)

### 1.1 状态定义

| 状态 | 代码 | 类型 | 客户可读标签 | 说明 |
|------|------|------|-------------|------|
| 待分配 | `open` | 初态 | 待处理 | 客户提交后默认进入 |
| 处理中 | `processing` | 中间态 | 处理中 | 客服领取/被指派后 |
| 待客户补充 | `waiting_customer` | 中间态（SLA 暂停） | 待我补充 | 客服请求客户提供更多信息 |
| 已解决 | `resolved` | 中间态 | 已解决 | 客服标记问题已解决 |
| 已关闭 | `closed` | **终态** | 已关闭 | 客服确认关闭，触发评价 |
| 已撤回 | `withdrawn` | **终态** | 已撤回 | 客户主动撤回（仅 open 状态可撤回） |
| 已合并 | `merged` | **终态** | 已合并 | 合并到主单，SLA 立即停止 |

### 1.2 流转矩阵

```
[客户提交] ──→ open
                 │
                 ├──→ processing（客服领取/指派）
                 │        │
                 │        ├──→ waiting_customer（请求补充）
                 │        │        │
                 │        │        └──→ processing（客户已补充）
                 │        │
                 │        ├──→ resolved（标记已解决）
                 │        │        │
                 │        │        └──→ closed（确认关闭 → 触发评价）
                 │        │
                 │        ├──→ closed（直接关闭）
                 │        │
                 │        └──→ merged（合并到主单）
                 │
                 └──→ withdrawn（客户撤回，仅 open 可达）
```

### 1.3 流转规则

| 来源状态 | 目标状态 | 触发条件 | 操作人 |
|----------|----------|----------|--------|
| `open` | `processing` | 客服领取或管理员指派 | 客服/管理员 |
| `open` | `withdrawn` | 客户主动撤回 | 客户 |
| `processing` | `waiting_customer` | 客服请求补充信息 | 客服 |
| `processing` | `resolved` | 客服标记已解决 | 客服 |
| `processing` | `closed` | 客服直接关闭 | 客服/管理员 |
| `processing` | `merged` | 合并到主单 | 客服/管理员 |
| `waiting_customer` | `processing` | 客户补充信息后自动恢复 | 系统 |
| `resolved` | `closed` | 确认关闭 | 客服/管理员 |

> **终态规则**：`closed`、`withdrawn`、`merged` 均不可重开。如需继续处理，必须新建工单并关联原单。

### 1.4 SLA 规则

| 状态 | SLA 时钟 |
|------|----------|
| `open` | 计时（首响 SLA） |
| `processing` | 计时（解决 SLA） |
| `waiting_customer` | **暂停** |
| `resolved` | 计时（等待关闭确认） |
| `closed` / `withdrawn` / `merged` | **终止** |

## 2. 咨询会话状态机 (`consultation_session.status`)

### 2.1 状态定义

| 状态 | 代码 | 类型 | 客户可读标签 | 说明 |
|------|------|------|-------------|------|
| 排队中 | `queued` | 初态 | 排队中 | 等待客服接入 |
| 进行中 | `active` | 中间态 | 进行中 | 客服已接入，实时通信 |
| 已结束 | `ended` | 中间态 | 已结束 | 会话正常结束（手动或超时） |
| 已归档 | `closed` | **终态** | 已关闭 | 评价完成或评价窗口到期 |

### 2.2 流转矩阵

```
[客户发起咨询] ──→ queued
                      │
                      └──→ active（客服接入）
                              │
                              ├──→ ended（客服手动结束 / 超时自动结束）
                              │       │
                              │       └──→ closed（评价完成 / 评价窗口到期）
                              │
                              └──→ [转接] active（转接后新客服接手，状态不变）
```

### 2.3 流转规则

| 来源状态 | 目标状态 | 触发条件 | 操作人 |
|----------|----------|----------|--------|
| `queued` | `active` | 客服手动接入 | 客服 |
| `active` | `active` | 转接（换客服，状态不变） | 客服 |
| `active` | `ended` | 客服手动结束 | 客服 |
| `active` | `ended` | 最后消息后 10 分钟无新消息 | 系统 |
| `ended` | `closed` | 客户完成评价 | 客户 |
| `ended` | `closed` | 评价窗口到期（3 小时） | 系统 |

> **转工单**：`active` 状态下客服可将会话转为工单，会话本身继续保持 `active`，不因转工单自动结束。

## 3. 反馈/建议状态机 (`ticket.status`，复用工单底座)

### 3.1 状态定义

| 内部状态 | 代码 | 客户可读标签 | 说明 |
|----------|------|-------------|------|
| 待评估 | `pending_review` | 已收到 | 客户提交后默认 |
| 处理中 | `in_progress` | 处理中 | 管理员领取并处理 |
| 已处理 | `completed` | 已完成 | 内部记录 `result`（`adopted`/`rejected`） |

### 3.2 流转矩阵

```
[客户提交] ──→ pending_review ──→ in_progress ──→ completed
```

### 3.3 流转规则

| 来源状态 | 目标状态 | 触发条件 | 操作人 |
|----------|----------|----------|--------|
| `pending_review` | `in_progress` | 客服/管理员领取 | 客服/管理员 |
| `in_progress` | `completed` | 处理完成，记录 `result` | 客服/管理员 |

> **补充说明**：客户仅在 `pending_review` 和 `in_progress` 阶段可追加补充，`completed` 后不可追加。

## 4. 安全告警状态机 (`security_alert.status`)

### 4.1 状态定义

| 状态 | 代码 | 说明 |
|------|------|------|
| 待处理 | `pending` | 告警触发后默认 |
| 已确认 | `confirmed` | 已认领并复核 |
| 已处置 | `handled` | 已采取修复/阻断/处理动作 |
| 已忽略 | `ignored` | 误报或无需处理 |

### 4.2 流转规则

| 来源状态 | 目标状态 | 触发条件 | 操作人 | 备注要求 |
|----------|----------|----------|--------|----------|
| `pending` | `confirmed` | 管理员认领 | platform_admin / 当前域 super_admin | 可选 |
| `confirmed` | `handled` | 采取处理动作 | platform_admin / 当前域 super_admin | **必填** |
| `confirmed` | `ignored` | 判定为误报 | platform_admin / 当前域 super_admin | **必填** |

## 5. 客户账号与入域状态机

### 5.1 客户全局账号状态定义 (`customer_account.status`)

| 状态 | 代码 | 说明 |
|------|------|------|
| 活跃 | `active` | 正常使用 |
| 禁用 | `disabled` | 管理员禁用 |
| 可恢复注销 | `cancelled_recoverable` | 客户发起注销，在恢复窗口内 |
| 终态注销 | `cancelled_final` | 恢复窗口结束，不可恢复 |

### 5.2 客户全局账号流转规则

| 来源状态 | 目标状态 | 触发条件 |
|----------|----------|----------|
| `active` | `disabled` | 管理员禁用 |
| `disabled` | `active` | 管理员重新启用 |
| `active` | `cancelled_recoverable` | 客户发起注销申请 |
| `cancelled_recoverable` | `active` | 恢复窗口内客户恢复 |
| `cancelled_recoverable` | `cancelled_final` | 恢复窗口到期（默认 7 天） |

### 5.3 客户入域状态定义 (`domain_customer.status`)

| 状态 | 代码 | 说明 |
|------|------|------|
| 待激活 | `pending` | 管理员手动添加后，客户尚未首次登录或激活当前域 |
| 活跃 | `active` | 客户可访问当前业务域 |
| 禁用 | `disabled` | 管理员在当前业务域禁用该客户 |

### 5.4 客户入域流转规则

| 来源状态 | 目标状态 | 触发条件 |
|----------|----------|----------|
| `pending` | `active` | 客户首次登录或完成注册激活当前域 |
| `active` | `disabled` | 管理员在当前域禁用 |
| `disabled` | `active` | 管理员在当前域重新启用 |

> 客户访问某业务域时，需同时满足 `customer_account.status=active` 与 `domain_customer.status=active`。

## 6. 知识条目状态机 (`knowledge_article.status`)

### 6.1 状态定义

| 状态 | 代码 | 说明 |
|------|------|------|
| 草稿 | `draft` | 编辑中，未提交审核 |
| 待审核 | `pending_review` | 已提交，等待审核 |
| 已发布 | `published` | 审核通过并发布 |
| 已下线 | `archived` | 主动下线或废弃 |

### 6.2 流转规则

| 来源状态 | 目标状态 | 触发条件 | 操作人 |
|----------|----------|----------|--------|
| `draft` | `pending_review` | 作者提交审核 | 知识编辑者 |
| `pending_review` | `published` | 审核通过 | 审核者 |
| `pending_review` | `draft` | 审核驳回 | 审核者 |
| `published` | `archived` | 主动下线 | 知识管理者 |
| `published` | `draft` | 重新编辑（生成新版本） | 知识编辑者 |
| `archived` | `draft` | 重新启用 | 知识管理者 |

## 7. 满意度评价状态机 (`satisfaction_rating.status`)

| 状态 | 代码 | 说明 |
|------|------|------|
| 待评价 | `pending` | 工单/会话关闭后自动生成 |
| 已评价 | `submitted` | 客户完成评价 |
| 已失效 | `expired` | 评价窗口到期（默认 3 小时） |

| 来源 | 目标 | 触发条件 |
|------|------|----------|
| `pending` | `submitted` | 客户提交评价 |
| `pending` | `expired` | 超过评价窗口 |
