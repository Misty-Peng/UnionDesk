# 通知映射退订与群发清单

## 业务域
- 通知中心

## 目标
- 将通知中心从“有渠道和重试”补充为“事件-模板-渠道映射、模板变量、邮件退订、非静默发送和批量群发”的完整可落地方案。

## 开发任务
- [x] 在 `3.9 通知中心` 中补充事件-模板-渠道默认映射表，明确账号安全、工单、咨询、反馈、知识和公告类事件的默认收件人和渠道。
- [x] 在 `3.9 通知中心` 中补充模板变量完整分组列表，明确全局、账号、工单、咨询、反馈、知识、公告和安全变量。
- [x] 在 `3.9 通知中心` 中补充邮件退订规则，明确非安全类邮件默认提供退订链接，安全类邮件不支持退订。
- [x] 在 `3.9 通知中心` 中明确不支持静默时间段或免打扰窗口配置。
- [x] 在 `3.9 通知中心` 中补充批量通知能力，明确域管理员可对本域用户群发、平台管理员可跨域群发。
- [x] 在 `5. 数据模型` 中补充 `notification_template` 与 `notification_broadcast_job`。
- [x] 在 `7. MVP 验收标准` 中补充映射、退订、静默、变量和群发验收项。

## 验证记录
- [x] 已确认通知中心采用“事件 -> 模板 -> 渠道 -> 收件人”映射表驱动。
- [x] 已确认安全类邮件不提供退订，非安全类邮件默认带退订链接。
- [x] 已确认通知中心不提供静默时间段配置，通知按事件实时发送。
- [x] 已确认批量通知支持域管理员本域群发、平台管理员跨域群发，并记录任务与发送日志。
- [x] 已确认模板变量按事件上下文注入，缺失变量不阻断发送。

- [x] 客户端普通通知采用未读数轮询，咨询 WebSocket 与普通通知通道分离
