# 权限变更 Token 失效清单

## 业务域
- Auth / 鉴权

## 目标
- 明确用户角色、业务域成员关系或账号状态变化后，已签发 Token 的失效时机与产品预期行为。

## 开发任务
- [x] 在安全章节中明确权限变更后访问令牌立即失效，刷新令牌同步失效。
- [x] 在 API 鉴权约定中补充 `auth_version` 口径，作为 Token 失效判断依据。
- [x] 明确实现可采用 Redis 版本号比对或令牌黑名单机制。

## 验证记录
- [x] 管理员修改用户角色后，旧 Token 立即失效。
- [x] 管理员移除业务域成员关系后，旧 Token 在该域内立即失效。
- [x] 管理员禁用账号后，旧 Token 立即失效，用户需重新登录。
