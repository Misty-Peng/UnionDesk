# 登录验证码与管理端接入开发清单

## 0. 文档说明
- [x] 本清单用于落实 AdminWeb 登录页滑块验证码、后端验证码校验、CustomerWeb 兼容和真实后端登录接入。
- [x] 验证码按 `auth_login_config.captcha_enabled` 全局生效，默认通过新增 Flyway 脚本开启。
- [x] 本轮只处理登录链路与必要兼容修复，不做无关页面重构。

## 1. 目标
- [ ] 后端提供滑块验证码挑战、轨迹校验和一次性验证码 token。
- [ ] `captcha_enabled=true` 时，登录接口强制消费有效验证码 token。
- [ ] AdminWeb 使用 `@uniondesk/shared` 真实登录客户端，不再依赖模板 mock 登录。
- [ ] CustomerWeb 在验证码开启后仍可正常登录。
- [ ] AdminWeb 与 CustomerWeb 均通过类型检查。

## 2. 开发任务
- [ ] 新增后端验证码服务测试，覆盖轨迹过短、速度异常、token 复用和登录成功消费。
- [ ] 新增后端验证码服务、控制器接口 DTO，并把验证码校验接入 `AuthService.login`。
- [ ] 追加 Flyway 脚本开启 `captcha_enabled` 与中文提示文案。
- [ ] 扩展 `packages/shared` 的登录类型、验证码 API helper 和 `SliderCaptcha` 后端校验能力。
- [ ] CustomerWeb 登录页按登录配置条件显示滑块，并提交 `captchaToken`。
- [ ] AdminWeb 包名改为 `admin-web`，接入 shared 登录、客户端标识和滑块验证码。
- [ ] 修复 AdminWeb 当前 typecheck 阻塞点，保持最小变更。

## 3. 验收标准
- [ ] 未完成滑块校验时，配置开启的登录请求被后端拒绝。
- [ ] 滑块校验成功后，登录请求携带一次性 token 并获得真实后端 session。
- [ ] 同一个验证码 token 不能重复登录。
- [ ] AdminWeb 与 CustomerWeb 均从 `packages/shared` 使用统一 API 类型。
- [ ] 不修改已发布 Flyway 脚本。

## 4. 本轮验证
- [ ] 后端定向测试：`.\mvnw.cmd -q "-Dtest=AuthServiceTests,AuthControllerTests,AuthCaptchaServiceTests" test`
- [ ] AdminWeb 类型检查：`pnpm --filter admin-web typecheck`
- [ ] CustomerWeb 类型检查：`pnpm --filter customer-web typecheck`
- [ ] 前端工作区类型检查：`pnpm -r typecheck`
- [ ] 项目提交前验证按 `AGENTS.md` §6 执行。
