# 数据范围与角色上限清单

## 0. 说明
- [x] 本清单用于收口业务域内数据范围、角色上限、权限项扩展与批量角色变更规则。

## 1. 设计收口
- [x] 数据范围仅保留系统预置的 `self` / `domain_all` 两档，不开放自定义条件表达式。
- [x] 数据范围适用于所有需要数据范围控制的模块，菜单 / 按钮权限与数据范围分层管理。
- [x] 每个业务域最多创建 20 个自定义角色，不包含系统预设角色。
- [x] 用户角色变更仅支持单个用户逐一调整，不支持批量角色变更。
- [x] 权限项由平台版本统一预置，域管理员不可自行新增权限项代码。

## 2. 数据模型
- [x] `domain_role_data_scope` 已移除为独立表，数据范围改由权限项后缀（`self` / `domain_all`）表达。
- [x] `domain_role_permission` 继续只承载菜单 / 按钮权限关系。

## 3. 验收
- [x] 域内自定义角色上限可控，超出 20 个时无法继续创建。
- [x] 角色数据范围只支持 `self` / `domain_all`，不可配置自定义条件。
- [x] 域管理员不能批量修改用户角色。
- [x] 新增模块权限项需由平台统一补充，域管理员仅能勾选已有权限项。
