# 账号模型收敛清单

## 业务域
- IAM / 身份与权限

## 目标
- 将 PRD 中员工账号、客户账号与统一身份主体的模型统一收敛为 `identity_subject` + `staff_account` + `customer_account`。
- 明确手机号在 MVP 阶段作为统一身份主体主匹配标识的边界。
- 明确业务表对员工账号、客户账号的引用语义，避免再使用单表 `account + account_type` 口径。
