# 稳定性与可用性清单

## 0. 说明
- [x] 本清单用于收口系统容量、恢复目标、缓存策略、日志保留、CDN 与告警规则。

## 1. 非功能目标
- [x] 性能目标已写入 PRD，首版容量不写死绝对并发数，以压测定容为准。
- [x] 月度可用性目标为 `99.9%`，计划维护窗口不计入。
- [x] 恢复目标为 `RTO 4` 小时、`RPO 24` 小时。

## 2. 基础设施策略
- [x] Redis 仅承担会话、验证码、`auth_version`、限流、热点配置、未读数和临时队列状态。
- [x] CDN 本期不启用，静态资源与附件先走源站 / 对象存储直链。
- [x] 应用运行日志与安全告警日志保留周期已明确，并统一落在固定目录。

## 3. 备份与恢复
- [x] MySQL 每日全量备份并归档 binlog。
- [x] 附件与帮助中心文件支持增量备份或版本保留。
- [x] 恢复流程为“数据库 -> 应用 -> 健康检查”。
- [x] 季度恢复演练纳入运维例行工作。

## 4. 监控告警
- [x] 监控接口错误率、延迟、数据库 / Redis 状态、备份结果、队列积压与关键安全事件。
- [x] 告警统一进入管理端安全/运维告警中心。
- [x] 告警规则支持后续按环境或阈值微调。
