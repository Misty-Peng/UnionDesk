# P0 验收用例表

> 本文档将 P0 验收标准转化为 Given/When/Then 格式的可执行用例，供 QA 和开发验证使用。
> 共 54 条用例（53 条 P0 + 1 条 P1 参考 TC-019），来源：`产品需求文档 v1.0.md § 7 MVP 验收标准` P0 对应编号。

---

## 1. 业务域管理

### TC-001：多业务域创建与隔离（验收 #1）

| 步骤 | 内容 |
|------|------|
| **Given** | 已登录 `platform_admin` 账号 |
| **When** | 分别创建 3 个业务域：A（`["public"]` + open）、B（`["domain_customer_only"]` + invitation_only）、C（`["domain_customer_only","channel_only"]` + admin_only） |
| **Then** | 3 个业务域均创建成功，各自可见策略与注册策略独立生效 |
| **And** | 域 A 客户可自注册并查看；域 B 未加入域客户无法看到；域 C 仅已入域客户或指定渠道客户可见且不可自注册 |
| **And** | 不同域内的工单、角色、配置互不可见 |

### TC-002：业务域直链访问（验收 #3）

| 步骤 | 内容 |
|------|------|
| **Given** | 已创建域 A（`["public"]` + open）和域 B（`["domain_customer_only"]` + invitation_only） |
| **When** | 客户通过直链访问域 A、域 B |
| **Then** | 域 A 直链可正常进入并触发注册/登录流程 |
| **And** | 域 B 直链对未邀请客户返回不可见提示 |

### TC-003：客户入域方式（验收 #4a）

| 步骤 | 内容 |
|------|------|
| **Given** | 已创建域 A（open）且生成邀请码 |
| **When** | 分别通过手动添加、自注册、邀请码三种方式入域 |
| **Then** | 每种方式入域成功，域内客户状态 `domain_customer.status` 为 `active`（自注册/邀请码）或 `pending`（管理员手动添加） |
| **And** | 禁用后无法登录该域，重新启用后恢复 |

## 2. 身份认证

### TC-004：客户端与员工端分别登录（验收 #2）

| 步骤 | 内容 |
|------|------|
| **Given** | 同一 `identity_subject` 下有员工账号和客户账号 |
| **When** | 先在员工端登录，再访问客户端 |
| **Then** | 客户端免重复登录，直接进入客户身份 |
| **When** | 仅有员工账号，访问客户端 |
| **Then** | 展示"无客户权限"提示页 |

### TC-005：登录安全防护（验收 #17）

| 步骤 | 内容 |
|------|------|
| **Given** | 已注册客户账号 |
| **When** | 连续输入 5 次错误密码 |
| **Then** | 第 6 次登录被拒绝，返回"账号已锁定，请 10 分钟后重试" |
| **When** | 等待 10 分钟后重试正确密码 |
| **Then** | 登录成功 |

### TC-006：Token 策略（验收 #27）

| 步骤 | 内容 |
|------|------|
| **Given** | 用户已登录，系统默认 Access Token 30 分钟 |
| **When** | Access Token 过期后使用 Refresh Token 刷新 |
| **Then** | 获得新的 Access Token 和 Refresh Token |
| **When** | 同一账号在第 6 个设备登录（超出默认并发 5） |
| **Then** | 最旧的会话被踢除 |

### TC-007：敏感操作二次验证（验收 #19）

| 步骤 | 内容 |
|------|------|
| **Given** | 已登录 `platform_admin` |
| **When** | 执行删除业务域操作 |
| **Then** | 系统要求重新输入密码 |
| **When** | 调用 `POST /api/v1/auth/step-up` 且密码正确 |
| **Then** | 返回 `step_up_token`，删除业务域接口需携带 `X-UD-Step-Up-Token` |
| **When** | 密码正确后 15 分钟内再次执行其他敏感操作 |
| **Then** | 不再要求二次验证（删除业务域除外，每次都需验证） |

## 3. RBAC 权限

### TC-008：角色与权限生效（验收 #12）

| 步骤 | 内容 |
|------|------|
| **Given** | 域 A 下创建自定义角色 R1，仅勾选 `ticket:menu` + `ticket:view` + `ticket:view:self` |
| **When** | 将员工 E1 加入域 A 并赋予角色 R1 |
| **Then** | E1 登录后仅可见工单菜单，仅能查看自己的工单，不可见咨询/知识库/域配置菜单 |
| **And** | E1 无法访问 `ticket:assign`、`ticket:reply` 等未授权接口 |

### TC-009：自定义角色上限（验收 #60）

| 步骤 | 内容 |
|------|------|
| **Given** | 域 A 已有 20 个自定义角色 |
| **When** | 尝试创建第 21 个 |
| **Then** | 系统拒绝并提示已达上限 |

### TC-010：权限变更即时失效（验收 #12 延伸）

| 步骤 | 内容 |
|------|------|
| **Given** | 员工 E1 已登录并持有有效 Token |
| **When** | 管理员移除 E1 的 `ticket:reply` 权限 |
| **Then** | E1 的下一次 API 请求因 `auth_version` 不匹配被拒绝，需重新登录 |

### TC-011：最后一个业务域 super_admin 保护（验收 #101）

| 步骤 | 内容 |
|------|------|
| **Given** | 域 A 中仅存 1 个激活的 `super_admin` |
| **When** | 尝试禁用、移除或降级该域级 `super_admin` |
| **Then** | 后端拒绝操作，提示"需先指定另一位业务域超级管理员" |

### TC-012：最后一个 domain_admin 保护（验收 #100）

| 步骤 | 内容 |
|------|------|
| **Given** | 域 A 中仅存 1 个 `domain_admin` |
| **When** | 该 `domain_admin` 尝试自行降级为 `agent` |
| **Then** | 后端拒绝，提示"需先指定另一位域管理员" |

### TC-053：最后一个 platform_admin 保护（验收 #111）

| 步骤 | 内容 |
|------|------|
| **Given** | 平台仅存 1 个激活的 `platform_admin` |
| **When** | 尝试禁用、删除、离职释放、降级，或通过 `/api/v1/admin/staff/{staff_id}/platform-roles` 回收该账号的平台管理员角色 |
| **Then** | 后端拒绝操作，提示"需先指定另一位平台管理员" |
| **When** | 先新增另一位激活的 `platform_admin` 后再回收原账号角色 |
| **Then** | 操作通过，审计日志记录授权变更主体、目标账号、动作和结果 |

## 4. 工单核心闭环

### TC-013：客户提交工单（验收 #5a）

| 步骤 | 内容 |
|------|------|
| **Given** | 客户已加入域 A，域 A 有工单类型"技术支持" |
| **When** | 客户在客户端选择"技术支持"类型，填写标题、描述并提交 |
| **Then** | 工单创建成功，状态为 `open`，生成工单编号 `{domain.code}-{日期}-{序号}` |
| **And** | 工单详情可查看，出现在"我的工单"列表 |

### TC-014：客服领取并处理工单（验收 #5a 延伸）

| 步骤 | 内容 |
|------|------|
| **Given** | 域 A 存在 `open` 状态工单 |
| **When** | 客服在工单队列中点击领取 |
| **Then** | 工单状态变为 `processing`，指派人变为当前客服 |
| **When** | 客服回复工单并标记已解决 |
| **Then** | 回复出现在时间线，状态变为 `resolved` |
| **When** | 客服确认关闭 |
| **Then** | 状态变为 `closed`，SLA 停止计时 |

### TC-015：客户撤回工单（验收 #5a 延伸）

| 步骤 | 内容 |
|------|------|
| **Given** | 客户提交的工单状态为 `open` |
| **When** | 客户点击撤回 |
| **Then** | 工单状态变为 `withdrawn`，不可重开 |
| **When** | 工单状态已为 `processing` 时尝试撤回 |
| **Then** | 撤回被拒绝 |

### TC-016：工单乐观锁冲突（验收 #5a 延伸）

| 步骤 | 内容 |
|------|------|
| **Given** | 两个客服同时打开同一工单详情 |
| **When** | 客服 A 先提交状态变更，客服 B 后提交 |
| **Then** | 客服 A 成功，客服 B 收到"工单已被他人修改，请刷新" |

### TC-017：工单状态流校验（验收 #5a 延伸）

| 步骤 | 内容 |
|------|------|
| **Given** | 工单状态为 `open` |
| **When** | 尝试直接变更为 `closed`（跳过 processing） |
| **Then** | 后端拒绝，返回"状态流转不合法" |

| 步骤 | 内容 |
|------|------|
| **Given** | 系统设置中员工端密码最小长度 10，客户端密码最小长度 8 |
| **When** | 员工设置 8 位密码，客户设置 8 位密码 |
| **Then** | 员工被拒绝（不满足 10 位），客户成功 |

## 17. 配置管理

### TC-045：系统配置与域配置（验收 #71）

| 步骤 | 内容 |
|------|------|
| **Given** | `platform_admin` 已登录 |
| **When** | 通过 `system_config` 修改平台级参数（如 Token 过期时间） |
| **Then** | 配置生效，新登录用户使用新 Token 时长 |
| **When** | 域管理员通过 `domain_config` 修改域级参数 |
| **Then** | 仅影响本域，其他域不受影响 |

### TC-046：快捷回复模板管理（验收 #72）

| 步骤 | 内容 |
|------|------|
| **Given** | 域 A 已创建快捷回复模板 |
| **When** | 客服在工单回复时选择快捷回复 |
| **Then** | 模板内容自动填充到回复框 |

## 18. 工单数据完整性

### TC-047：工单操作历史（验收 #81）

| 步骤 | 内容 |
|------|------|
| **Given** | 客户提单、客服领取、客服回复、系统自动变更状态 |
| **When** | 查看工单 `ticket_history` |
| **Then** | 每条记录包含 `operator_subject_id` 和 `operator_actor_type`，可区分客户/客服/系统 |

### TC-048：工单回复时间线（验收 #99）

| 步骤 | 内容 |
|------|------|
| **Given** | 工单包含多条公开回复、内部备注和系统记录 |
| **When** | 查看工单详情 |
| **Then** | 回复按 `created_at` 时间线排序展示，各类型可识别 |

### TC-049：审计日志域筛选（验收 #98）

| 步骤 | 内容 |
|------|------|
| **Given** | 平台级操作和域 A 操作均已产生审计日志 |
| **When** | 按域 A 筛选审计日志 |
| **Then** | 仅返回域 A 的记录，平台级操作（domain_id=NULL）不在其中 |

## 19. 环境与架构约束

### TC-050：环境隔离（验收 #26）

| 步骤 | 内容 |
|------|------|
| **Given** | 开发、测试、预发布、生产 4 套环境均已部署 |
| **When** | 在测试环境执行操作 |
| **Then** | 生产环境数据、配置、访问入口均不受影响 |

### TC-051：Redis 用途限定（验收 #64）

| 步骤 | 内容 |
|------|------|
| **Given** | 审查后端代码 Redis 使用场景 |
| **When** | 检查所有 Redis 调用点 |
| **Then** | 仅用于会话、auth_version、验证码、限流、热点配置、未读数、临时队列，无核心业务数据存储 |

### TC-052：可用性与恢复（验收 #65）

| 步骤 | 内容 |
|------|------|
| **Given** | P0 单机部署环境已配置备份脚本并准备恢复手册 |
| **When** | 执行一次恢复流程检查 |
| **Then** | 健康检查、备份脚本、恢复步骤与 RTO/RPO 目标均已文档化；数据库主从和季度恢复演练作为生产上线 Gate 不阻塞 P0 |
