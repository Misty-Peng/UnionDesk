## 1. 产品概述

### 1.1 产品定位
UnionDesk 是一个企业级、多业务域、可配置的客户服务中台。它统一管理**工单、在线咨询、反馈/建议、知识库**四类请求，通过业务域实现不同品牌/业务线的独立运营与权限隔离。

### 1.2 项目边界
- **后端（Spring Boot）**：API 服务、鉴权/授权、业务规则、通知引擎、审计日志。
- **前端（React）**：两个独立 SPA 应用 —— 客户端（用户端）与管理端（内部工作台）。
- **MVP 不包含**：AI 自动回复、可视化 BI 报表、外部供应商协作门户。
- **短信通道**：MVP 不内置短信发送网关；系统架构预留短信渠道接口，部署方可选接入第三方短信网关。未接入时，找回密码、手机号改绑等场景自动降级为邮箱验证码，邮箱也不可用时转管理员处理（详见 1.5 外部依赖确认表）。

### 1.3 MVP 核心价值
- 多业务域完全隔离，独立品牌入口与可见策略。
- 客户端与员工端分离登录，统一身份主体下可共享认证会话。
- 动态工单表单引擎，域管理员低代码自定义字段和状态流。
- 完整服务闭环：在线咨询、工单流转、知识库辅助、SLA 保障。
- 关键字安全屏蔽与全链路操作审计。

## 2. 身份与权限

### 2.1 身份模型与角色分层

#### 2.1.1 核心原则
**统一身份主体 + 角色分层 + 作用域隔离**：
- 平台角色和业务域角色分属不同体系，互不污染。
- 系统以 `identity_subject` 作为统一身份主体，一个自然人只归属一个主体。
- 一个主体下可以同时挂载员工账号和客户账号。
- 客户端默认进入客户身份，员工端默认进入员工身份，不设置独立统一入口页。
- 两个端分别登录，但底层可共享认证会话；同一主体切换到另一端时，如已具备对应账号与授权，则免重复登录。
- 若当前主体没有对应账号或无权限，则保留登录态并提示无权限。

#### 2.1.2 账号模型
- 身份模型采用 **1 个统一身份主体 + 2 张账号扩展表**：`identity_subject` 表示自然人身份主体，`staff_account` 表示员工账号，`customer_account` 表示客户账号。
- `identity_subject.phone` 是 MVP 阶段统一身份主体的主匹配标识，并设置唯一约束；同一手机号默认归并为同一个 `identity_subject`。
- **员工账号**：`staff_account`，通过 `subject_id` 关联 `identity_subject`，用于员工端、平台管理和业务域管理。
- **客户账号**：`customer_account`，通过 `subject_id` 关联 `identity_subject`，用于客户端、工单提报、咨询、知识访问和评价。
- 一个 `identity_subject` 可以同时拥有一个员工账号和一个客户账号；员工账号与客户账号的登录手机号必须与 `identity_subject.phone` 保持一致，如需保存其他联系方式，联系手机号不作为登录手机号。
- 当员工账号和客户账号使用不同手机号创建时，系统会创建两个独立的 `identity_subject`。后续如确认为同一自然人，由 `platform_admin` 通过“身份归并”功能手动合并，合并后被合并主体的 `merged_into_id` 指向保留主体；保留主体自动选择历史存在身份对应的手机号作为登录手机号，其他联系方式仅作为联系信息保存。身份归并不迁移历史工单、咨询和审计日志等事实记录，仅在查询与展示时按 `merged_into_id` 归并为同一自然人视图。
- 客户账号来源包括：内部员工手动创建、自注册、邀请码注册、云之家单点登录自动注册/绑定。
- 一个客户账号可加入多个业务域；每个业务域独立维护激活状态、标签、服务记录和可见权限。
- 手动添加客户默认处于`待激活`状态，首次登录或注册后自动激活。
- 客户从某个业务域移除时，等同于该客户在该域内被禁用，不影响全局客户账号和其他业务域关系。
- 云之家首次接入优先按唯一标识匹配统一身份主体；若手机号、邮箱等高可信字段可以唯一确认同一自然人，则自动归并到同一主体，不再提示用户。
- 若无法唯一确认，进入后台人工处理，不做自动绑定。
- `identity_subject.subject_type` 仅用于区分主体是`person`还是`system`，不区分员工/客户；当前 MVP 仅实际使用`person`，`system`为预留扩展位，不参与门户与权限判断。

#### 2.1.3 平台角色（全局表 `platform_role`）

| 角色 | 代码 | 是否预设 | 说明 |
|------|------|----------|------|
| 平台管理员 | `platform_admin` | 是，可授权多用户 | 平台级最高权限，负责业务域创建、平台配置、全局用户授权和跨域治理 |

- 平台角色**不可由业务域管理员增删改**，由系统预设。
- `platform_admin` 权限高于任一业务域内的 `super_admin`，可创建业务域、管理平台级配置，并可在业务域内授予或回收域级 `super_admin`。
- `platform_admin` 创建业务域时，系统自动将创建者加入当前业务域并授予该业务域的 `super_admin` 角色，写入审计日志；该自动授权用于让创建者立即接管当前域，不表示其获得其他业务域内容权限。
- 平台必须始终保留至少 1 个激活的 `platform_admin`；最后一个激活的 `platform_admin` 不允许被禁用、删除、离职释放、降级或回收平台角色。如发生初始化配置错误，需通过平台初始化脚本或人工运维流程恢复，并写入审计日志。

#### 2.1.4 业务域角色（域内表 `domain_role`）

每个业务域默认内置三个固定角色，域管理员可在此基础上创建自定义角色；自定义角色最多 20 个 / 域，不含系统预设角色。

| 角色 | 代码 | 是否预设 | 典型权限 |
|------|------|----------|----------|
| 域超级管理员 | `super_admin` | 是，不可删除 | 当前业务域最高权限，可管理本域配置、成员、角色、权限与域级审计 |
| 域管理员 | `domain_admin` | 是，不可删除 | 管理本域常规配置、用户、角色、权限，不可授予或回收 `super_admin` |
| 客服 | `agent` | 是，不可删除 | 处理工单、接待咨询、使用知识库 |

- 预设角色的核心权限由系统内置，不可被域管理员更改或删除，但可被**复制**后修改为新角色。
- 每个业务域必须始终保留至少 1 个激活的 `super_admin`；最后一个 `super_admin` 不允许被降级、禁用或移除。
- 域管理员可在本域内创建自定义角色（详见 2.1.5）。

#### 2.1.6 权限粒度：菜单级与按钮级
平台支持**双层权限控制**：
- **菜单级**：控制用户是否能看到某个功能菜单 / 页面入口。前端路由守卫 + 后端接口返回菜单清单。
- **按钮级（操作级）**：控制用户在页面内能否看到或执行具体按钮/操作。前端根据权限项清单隐藏/禁用按钮；后端在每个操作接口上进行权限项校验。

**实现方案**：
- 前端：封装鉴权组件（如 `<Permission code="ticket:delete">`），包裹需要控制的按钮或区域。无权限时元素不渲染或置灰。
- 后端：自定义注解（如 `@RequirePermission("ticket:delete")`）结合 Spring Security 或 AOP 拦截器校验当前用户域内权限项。
- 权限项设计遵循 `模块:操作` 或 `模块:菜单` 命名规范，清晰可读。

### 2.2 系统可见范围
| 角色 | 可见范围 |
|------|----------|
| `platform_admin` | 平台管理后台（全部功能）；可创建业务域、管理平台配置、进行全局授权和跨域治理；创建业务域时自动成为当前域 `super_admin` |
| `super_admin` | 所属业务域的最高权限角色；可进入对应业务域并默认拥有该域全部权限，不具备平台级全局配置权限 |
| `domain_admin` | 所管业务域的管理后台 |
| `agent` / 自定义角色 | 所属业务域的工作台（按权限项显示） |
| `customer` | 客户端界面 |

## 3. 功能需求

### 3.1 通用访问与登录规则

#### 3.1.1 客户端与员工端入口
- 客户端入口默认进入客户身份，员工端入口默认进入员工身份。
- 不设置独立统一入口页；客户端和员工端分别登录，但底层可共享认证会话。
- 同一统一身份主体在已登录状态下访问另一端时，如已具备对应账号与授权，则免重复登录。
- 若当前主体没有对应账号或无权限，则保留登录态并提示无权限。

#### 3.1.2 业务域直链访问
- **场景**：用户通过浏览器直接访问某个业务域的专属 URL（如 `https://domain-a.uniondesk.com` 或 `https://uniondesk.com/d/domain-a`）。
- **未登录状态**：
  - 系统检测到用户未登录，重定向至当前门户登录页，并在 URL 参数中携带目标业务域标识（如 `?redirect=/d/domain-a`）。
- **登录成功后**：
  - 系统根据 URL 参数中的目标业务域标识，跳过业务域选择步骤，直接进入该业务域首页。
  - 若用户对该业务域无访问权限，则展示无权限提示页，并引导用户切换到其他可访问业务域或联系管理员。
- **已登录状态**：
  - 直接进入目标业务域首页，无需再次选择。
- **域禁用状态**：
  - 业务域禁用后，客户端整体不可见，包含该域公开知识。

#### 3.1.3 业务域可见策略与注册策略

业务域的**可见策略**和**注册策略**是两个独立维度，分别配置。

##### 可见策略（`visibility_policy`）
- 采用可组合规则，不是单值枚举。
- 规则编码采用稳定英文代码，并以数组形式保存；P0 接口统一使用 `visibility_policy_codes`。

| 规则 | 代码 | 说明 |
|------|------|------|
| 公开 | `public` | 匿名用户可访问公开入口与公开知识 |
| 仅当前域客户列表可见 | `domain_customer_only` | 只有已加入当前域的客户可访问 |
| 仅渠道可见 | `channel_only` | 只能从指定渠道入口访问 |

- `public` 与后两项互斥；`domain_customer_only` 与 `channel_only` 可以同时启用，满足任一条件即可访问。
- 匿名用户可访问公开知识库，并可搜索与查看公开域中的公开知识。
- `仅当前域客户列表可见` 的域中，只有已加入当前域的客户可访问。
- `仅渠道可见` 的域中，客户只能从指定渠道进入。
- 公共知识库的已发布公开条目支持未登录用户访问；业务域公开知识仅在对应业务域状态为启用时可访问。

##### 注册策略（`registration_policy`）
- 控制客户能否自主注册加入业务域，独立于可见策略配置。
- 取值：

| 策略 | 代码 | 说明 |
|------|------|------|
| 开放注册 | `open` | 允许客户自主注册并加入当前域 |
| 仅邀请注册 | `invitation_only` | 客户仅可通过邀请码/邀请链接注册加入 |
| 仅管理员添加 | `admin_only` | 禁止自注册，仅管理员手动添加客户 |

- 渠道入口（`仅渠道可见`）中，若注册策略为 `open`，未注册客户从渠道进入时自动注册并加入当前域；若为 `invitation_only` 或 `admin_only`，渠道入口仅对已注册客户可见。
- 典型组合示例：
  - `公开` + `open`：公开域，任何人可自注册（最宽松）。
  - `公开` + `invitation_only`：公开可见但需邀请码才能注册。
  - `仅渠道可见` + `open`：渠道入口自动注册。
  - `仅客户列表可见` + `admin_only`：纯管理员管控（最严格）。

#### 3.1.4 客户入域与激活
- 客户入域方式受业务域 `registration_policy` 控制：
  - `open`：客户可通过域入口自主注册并加入。
  - `invitation_only`：客户仅可通过邀请码/邀请链接注册加入。
  - `admin_only`：禁止自注册，仅管理员手动添加。
- 手动添加客户后，客户域内状态默认为`待激活`。
- 客户首次登录或注册后自动激活。
- 若手动添加后邀请链接失效且客户尚未登录，域内状态保持`待激活`，管理员可重发邀请。
- 邀请码仅用于注册进入，支持有效期和最大使用次数限制；当 `registration_policy` 为 `admin_only` 时邀请码功能不可用。
- 邀请码注册成功后，自动标记客户来源为邀请码渠道。
- 客户从某个业务域移除时，等同于该客户在该域内被禁用，不影响其他业务域关系。

#### 3.1.5 云之家单点登录与身份归并
- `仅渠道可见` 的域支持云之家单点登录。
- 云之家唯一标识只能绑定到一个统一身份主体。
- 若云之家首次接入时，手机号、邮箱等高可信字段可以唯一确认与已有员工账号或客户账号属于同一自然人，则自动归并到同一身份主体，不再提示用户。
- 若同一客户账号已存在于其他业务域，允许复用该客户账号并自动加入当前域。
- 若无法唯一确认，进入后台人工处理，不做自动绑定。
- 身份归并仅调整统一身份主体关系，不迁移历史工单、咨询和审计日志等事实记录；原始引用保留，查询与展示时按 `merged_into_id` 归并为同一自然人视图。

#### 3.1.6 会话恢复
- 用户关闭浏览器后重新打开，若 Token 仍有效，直接恢复上次访问的页面上下文，不重复登录。

#### 3.1.7 账号安全与自助管理
- **验证码与登录风控**：登录、注册、找回密码默认启用验证码/滑块校验；登录连续失败 3 次后升级为强校验，单 IP `5` 次 / `10` 分钟限流，单账号 `5` 次 / `10` 分钟锁定 `10` 分钟。
- **找回密码**：客户端和员工端共用找回密码流程。优先通过短信验证码验证；当系统未配置短信通道或当前账号未绑定可用手机号时，自动改走邮箱验证码验证；短信和邮箱均不可用时，转由管理员重置；单账号每日最多 3 次。
- **修改密码**：已登录用户可修改自己的密码；修改成功后，旧 Token 立即失效并要求重新登录。
- **客户自助资料**：客户可修改头像、昵称等展示型资料；`identity_subject.phone` 不作为普通资料项直接修改。
- **绑定手机号/邮箱**：
  - 手机号改绑仅在短信通道可用时开放，必须先验证旧手机号与新手机号；成功后同步更新该统一身份主体下所有账号的登录手机号。若未配置短信通道，前端直接隐藏/置灰该入口，后端返回“请联系管理员修改”。
  - 邮箱改绑支持邮箱验证码验证；若邮箱通道不可用且无法完成二次验证，由管理员处理。
- **首次登录强制改密**：手动创建、管理员重置或发放临时密码的员工/客户账号，首次成功登录后必须先修改密码，再进入业务页面；该状态通过 `must_change_password` 标记控制。自助找回密码完成的新密码不再强制二次修改。
- **账户注销**：客户可发起自助注销申请；注销仅作用于 `customer_account` 及其业务域关系，不影响同一 `identity_subject` 下的员工账号。注销完成后停止登录、撤销 Token、关闭仍在进行中的咨询会话，未关闭工单继续保留并由客服处理，历史工单/咨询/审计记录保留并按合规要求做匿名化处理。员工账号不开放自助注销，由平台管理员/域管理员通过离职池处理。
- **账户恢复**：客户账号注销审核通过后进入 `cancelled_recoverable` 状态，默认保留 7 天恢复窗口，可配置；在窗口期内，客户可通过短信验证码优先、邮箱验证码兜底的方式恢复账号，短信和邮箱均不可用时由管理员人工恢复。恢复成功后账号回到 `active`，原有业务域关系恢复，旧 Token 继续失效并要求重新登录；恢复窗口结束后进入 `cancelled_final`，不再支持恢复。
- **隐私政策与个人信息处理规则**：客户端提供独立页面展示隐私政策/个人信息处理规则，明确收集目的、保存期限、共享范围、删除/注销方式；注册或首次登录时需确认同意，政策版本更新后保留历史版本与更新时间。
- **Token 策略**：Access Token 默认有效期 `30` 分钟，Refresh Token 默认有效期 `7` 天；同一账号允许最多 `5` 个并发会话，按账号维度分别统计；Refresh Token 默认启用轮换机制，每次刷新都废弃旧 Refresh Token，超出并发上限时自动踢掉最旧会话；所有参数均可通过全局配置调整。
- **权限边界**：找回密码、修改密码、修改资料、改绑联系方式、注销申请等自助能力不纳入 `permission_item`，仅校验账号所有权与当前认证状态。

### 3.2 平台管理（仅 `platform_admin` 可见）
- `platform_admin` 是平台级最高权限，可进行业务域实体管理、平台配置、全局用户授权和跨域治理；不默认拥有任何既有业务域内容访问权，如需处理域内内容，必须先被加入对应业务域并授予域内角色。
- **平台管理员保活**：平台必须始终保持至少 1 个激活的 `platform_admin`；当尝试禁用、删除、离职释放、降级或回收最后一个 `platform_admin` 时，后端直接拒绝并提示“请先指定另一位平台管理员”。
- **域超级管理员保活**：每个业务域必须始终保持至少 1 个激活的 `super_admin`；当尝试禁用或移除某域最后一个 `super_admin` 时，后端直接拒绝并提示“请先指定另一位业务域超级管理员”。

#### 3.2.0 平台工作台首页
- **默认入口**：`platform_admin` 登录后默认进入平台工作台首页；从域工作台可通过“平台管理”入口切换进入平台工作台或平台管理页。
- **核心卡片**：全平台业务域总数、正常运行业务域数、当前在线客服总数、当前排队客户总数、待处理工单数、今日新建工单数、今日关闭工单数、待处理安全告警数、归档/导出任务进行中数量。
- **趋势图**：本周工单新增/关闭趋势、本周咨询排队趋势、本周安全告警趋势、本周归档/导出任务趋势。
- **快捷入口**：业务域管理、安全告警中心、通知群发、数据归档/导出、权限与角色管理、系统设置。
- **设计边界**：仅提供数字卡片与简易趋势图，不作为 BI 报表系统，不提供复杂钻取分析。

#### 3.2.1 业务域管理
- **业务域列表**：展示所有业务域及其状态（启用/禁用）、可见策略、域管理员、创建时间。
- **新建业务域**：支持两种方式——
  - **从模板新建**：内置“默认业务域”模板，预置工单类型（问题）、反馈、投诉。
  - **空白新建**：从零创建。
- **创建后授权**：`platform_admin` 新建业务域成功后，系统自动将创建者加入该业务域并授予当前域 `super_admin`，写入审计日志，便于立即接管新域。
- **编辑/删除**：修改业务域信息，删除为软删除；软删除后日常列表隐藏，域成员不可继续访问该域内容，但 `platform_admin` 仍可通过历史查询与导出入口只读查看该域历史工单、咨询、知识和审计记录；原域 `super_admin` 可在历史授权范围内只读查看本域历史数据。
- **复制业务域**：一键复制某个业务域的全部配置（工单类型、状态流、SLA 规则、自定义字段、通知模板、知识库结构、自定义角色等），不复制数据（工单、咨询记录、用户）。新域 code 需手动填写。
- **提炼为模板**：将任一个业务域的当前配置提炼为模板，供新建时复用。
- **模板管理**：模板内容由 `domain_template` 持久化，支持模板名称、模板类型、来源业务域、配置快照和启用状态。
- **启用/禁用**：禁用后客户端不可见，已在途工单不受影响，新工单不可提交，公开知识也不可访问。

#### 3.2.2 员工账号管理
- **用户列表**：展示所有内部员工账号及其状态、平台角色、所属业务域、角色。
- **添加用户**：手动创建账号或批量导入 CSV。
- **批量导入 CSV**：模板采用 UTF-8 CSV，首行为表头；字段建议为 `登录名`（必填）、`手机号`（必填）、`邮箱`（可选）、`平台角色`（可选）、`业务域编码`（可选）、`业务域角色`（可选）、`启用状态`（可选）、`初始密码`（可选）。至少填写 `平台角色` 或 `业务域编码 + 业务域角色` 之一；登录名、手机号与角色编码需校验唯一性与存在性；业务域编码必须有效。导入支持部分成功：成功行入库、失败行生成错误报告下载，不做全量回滚，导入任务写入 `data_import_job`。
- **编辑/删除**：管理员可修改员工账号信息。
- **角色配置流程**：
  - 在平台管理侧，需**先选择域上下文**再分配角色：
    - **选择平台域** → 可为该用户分配/修改平台角色（`platform_admin`）。
    - **选择某个业务域** → 可查看并分配该业务域内的角色（包括 `super_admin`、`domain_admin`、`agent` 及自定义角色）。
  - 一个内部员工账号可同时在平台域拥有角色，并在多个业务域拥有角色。
- **离职池**：删除员工账号后进入“离职池”，保留历史操作记录和关联数据，同时释放登录权限和所有域成员席位；该员工名下所有业务域中的未关闭工单自动解除指派并回退至本域未分配队列，保留历史指派记录并通知域管理员接管，域管理员可批量再分配。`platform_admin` 可手动彻底删除或恢复。

#### 3.2.3 业务域用户/客户列表
- 查看每个业务域下的内部成员目录和客户目录（基础信息），支持按域筛选、按激活状态筛选；目录信息仅用于授权与运维，不视为域内容。
- 支持跨域添加/移除成员关系。

#### 3.2.4 平台角色管理
- 管理 `platform_admin` 角色的授权用户；新增、回收、禁用、删除或离职释放 `platform_admin` 均属于敏感操作，需二次验证并写入审计日志。
- 若操作会导致平台无激活 `platform_admin`，后端必须拒绝，不允许通过批量导入、离职池、账号禁用、角色回收等旁路绕过。
- `platform_admin` 可对业务域内其他用户授予/回收该域 `super_admin`；手动授予/回收需通过敏感操作二次验证并写入审计日志。

#### 3.2.5 公共知识库管理
- 创建/编辑/删除公共知识库。
- 指定公共知识库维护人员（跨域）。

#### 3.2.6 日志与审计
- **操作日志**：全局管理操作审计，支持按操作人、业务域、时间、操作类型筛选。
- **登录日志**：所有用户的登录记录（成功/失败、IP、时间）。
- **登录日志持久化**：登录成功/失败记录统一写入 `login_log`，支持按账号、门户、IP、时间和结果筛选。
- **运行日志保留**：应用运行日志默认保留 `180` 天，安全告警日志默认保留 `365` 天；日志文件统一落在固定目录，不在其他位置分散生成。

#### 3.2.7 系统设置
- **全局安全策略**：密码复杂度要求按 `portal_type` 区分（`staff` / `customer` 可分别配置）、验证码策略、登录失败锁定阈值（单 IP `5` 次 / `10` 分钟、单账号 `5` 次 / `10` 分钟锁定 `10` 分钟）、会话超时时间、敏感操作密码复核窗口（默认 `15` 分钟，可配置）、客户注销恢复窗口（默认 7 天，可配置）。
- **全局风控阈值**：登录、找回密码、咨询发消息、工单创建、附件上传等接口的限流阈值与告警规则，可按业务域覆盖。
- **员工端 IP 访问控制**：员工端与平台管理后台支持 IP 白名单，默认关闭，可按平台或业务域配置；命中白名单外 IP 时拒绝登录，并记录 `login_log` 与 `security_alert`，客户端不受影响。
- **归档与导出入口**：数据归档动作由平台管理后台的数据治理/归档中心触发，仅 `platform_admin` 可见；数据导出可由工单、客户、审计日志列表页按钮触发，也可在平台管理后台的统一导出中心触发，均需对应权限项并写入审计日志。
- **全局数据保留策略**：工单、咨询消息、审计日志保留周期默认 `3` 年，可配置；超过热数据窗口的历史数据自动归档为只读。
- **全局归档策略**：热数据窗口默认 `12` 个月，可配置；归档后仍可通过专用查询入口按工单号、客户、时间和业务域精确检索，不进入默认列表。
- **全局导出策略**：支持工单、客户和审计日志基础导出，导出任务异步生成，默认提供 CSV / XLSX，敏感字段脱敏，导出文件默认有效期 `7` 天，导出行为写入审计日志。
- **发布控制**：功能开关与灰度范围可配置，支持按业务域、角色、白名单控制新功能启停；预发布与生产发布需走审批流程。
- **全局通知配置**：SMTP 服务器配置、短信网关配置（可选）、默认邮件模板；短信网关启用后可用于找回密码与手机号改绑。
- **隐私政策管理**：平台统一维护隐私政策/个人信息处理规则版本、正文和生效时间，客户端与注册流程引用同一版本。
- **全局关键词屏蔽**（可选）：平台级敏感词库，所有业务域生效（与域级词库叠加）。
- **附件存储策略**：存储方式（本地/对象存储）、大小限制（默认单文件 `200M`）、允许的文件类型（平台默认提供图片、文档、压缩包等常见格式，域管理员可继续增补）。
- **附件统一管理**：工单、咨询、知识等业务对象统一通过 `file_attachment` + `attachment_ref` 关联附件，附件策略仅控制类型、大小和存储方式，不分别为各模块重复配置。
- **平台全局配置**：平台级可变参数统一通过 `system_config` 维护，适合开关、默认值、阈值等 KV 配置。
- **Token 策略配置**：Access Token TTL、Refresh Token TTL、最大并发会话数、Refresh Token 轮换开关与并发会话踢除策略统一通过 `system_config` 维护，默认值分别为 `30` 分钟、`7` 天、`5` 个会话、启用轮换、踢除最旧会话。
- **全局缓存策略**：Redis 仅用于登录会话、`auth_version`、验证码、限流计数、热点配置、未读数和临时队列状态，不作为核心业务主存储。
- **全局备份与恢复策略**：P0 要求提供单机部署下的数据库备份脚本、附件备份说明、恢复手册与一次恢复流程验证；MySQL 主从、附件增量备份自动化和季度恢复演练作为生产上线 Gate，不阻塞 P0 开发验收。RTO 默认 `4` 小时、RPO 默认 `24` 小时作为恢复目标记录，可随部署环境微调。
- **全局可用性目标**：核心服务月度可用性 `99.9%`；计划维护窗口需提前公告，并不计入月度可用性。
- **全局监控告警**：监控接口错误率、延迟、数据库 / Redis 状态、备份结果、队列积压、SLA 达成率与关键安全事件；告警规则与通知方式统一由管理端安全/运维告警中心承载。
- **全局 CDN 策略**：本期不启用 CDN；静态资源与附件先按源站 / 对象存储直链访问，后续预留 CDN 接入能力。
- **全局系统配置**：平台级通用参数统一通过 `system_config` 维护；业务域零散参数统一通过 `domain_config` 维护，专用结构化配置继续保留在各自配置表中。

### 3.3 业务域管理（域管理后台，仅 `super_admin`、`domain_admin` 及拥有管理权限的自定义角色可见）

#### 3.3.0 域工作台首页
- **默认入口**：`super_admin`、`domain_admin` 及具备域内管理权限的角色登录后默认进入域工作台首页；`platform_admin` 若已被加入该业务域并授予域内角色，也可查看该域工作台首页。
- **核心卡片**：今日新建工单数、今日关闭工单数、当前待处理工单数、当前在线客服数、当前排队客户数、SLA 首响达标率、SLA 解决达标率、本周反馈数量。
- **趋势图**：本周工单新增 / 关闭趋势、本周反馈趋势、SLA 达标率趋势。
- **快捷入口**：工单管理、咨询接待、反馈/建议、知识库管理、客服在线状态、域配置。
- **设计边界**：仅提供运营工作台级别的数字卡片与简易趋势图，不作为 BI 报表系统，不提供复杂钻取分析。

#### 3.3.1 域基本信息
- **增删改查**：修改业务域名称、Logo、可见策略、注册策略。
- **域管理员保活**：每个业务域至少保留 1 个 `domain_admin`；若某次移除或降级操作会导致该域没有任何 `domain_admin`，后端直接拒绝并提示“请先指定另一位业务域管理员”。
- **复制配置**：复制本域全量配置到另一个新域，不复制数据。
- **提炼模板**：将本域配置提炼为模板供平台复用。
- **通用参数**：除主题色、工作日历等专用配置外，域级零散参数统一通过 `domain_config` 承载。

#### 3.3.2 工单配置
- **工单类型管理**：创建/编辑/启停工单类型，配置动态字段、状态流。
- **SLA 规则配置**：配置本域默认 SLA 及按类型覆盖规则。
- **工作日历**：配置本域的工作日、节假日、自定义工作时段。
- **状态流编辑器**：可视化编辑状态流图（上限 10 状态），系统自动检测孤立状态并警告。
- **附件策略配置**：域管理员可在本域管理后台的“附件策略”入口覆盖平台默认附件规则，配置允许类型、单文件大小和总大小上限；工单、咨询、知识统一复用同一 `attachment_policy`，其中 `max_total_size_mb` 按单个业务对象维度生效（单工单 / 单会话 / 单知识条目分别累计），优先级为域级覆盖平台默认值。
- **附件上传流程**：默认使用 MinIO 作为对象存储，上传采用客户端直传对象存储方式，API 负责鉴权、生成预签名上传凭证与落库；若 Sprint 0 未确认 MinIO，则启用本地文件存储模式，走 API multipart 上传并落库 `storage_type=local`。本地模式由后端配置 `storage.local_root`，对外不暴露服务器绝对路径，附件访问统一通过鉴权下载接口或短时签名 URL；本地文件需纳入备份清单，清理任务只能删除无业务引用的临时/孤儿文件，后续通过存储迁移任务切换到对象存储并保持 `attachment_id` 不变。

#### 3.3.3 域内用户管理
- **用户列表**：展示本域内所有内部成员及其状态、角色。
- **添加用户**：从平台已有内部账号中添加，或手动创建新内部账号并直接加入本域。
- **角色分配**：在业务域管理侧，直接为所选用户配置**本业务域内的角色**（无需再选域），支持分配一个或多个角色。
- **离职池（域级）**：移除域内用户时进入域离职池，仅释放本域权限，不影响其他域；该用户在本域中的未关闭工单自动解除指派并回退至本域未分配队列，保留历史指派记录并通知域管理员接管。
- **批量角色变更**：不支持批量修改用户角色，仅支持单个用户逐一调整。

#### 3.3.4 域内客户列表
- 展示本域客户列表，支持按激活状态、注册时间、手机号、标签搜索。
- 支持管理员手动激活/禁用客户，并维护客户标签。
- 当业务域 `registration_policy` 为 `admin_only` 时，自注册入口不可用，客户仅可通过此列表由管理员手动添加。

#### 3.3.5 业务域角色与权限管理
- **角色列表**：展示本域所有角色（含系统预设及自定义）。
- **新建/编辑/删除自定义角色**：
  - 填写角色名称、编码，勾选所需权限项（按模块分组展示，区分菜单权限和操作权限）。
  - 可基于预设角色复制创建，快速生成近似角色。
  - 预设角色不可编辑/删除，仅可查看其权限项。
- **权限项列表**：系统预置权限项（见 2.1.5），分为菜单级和按钮级；涉及数据范围的权限项通过后缀区分（如 `self` / `domain_all`），域管理员仅能组合勾选。
- **用户角色分配**：在用户管理中可将一个或多个角色分配给用户。
- **权限项来源**：权限项由平台统一发布和维护，域管理员只能勾选已发布的权限项，不可自行新增权限项代码；数据范围通过权限项后缀表达，不再单独提供配置面板。

#### 3.3.6 知识库管理
- 管理本域知识库（创建/编辑/删除/可见范围设置），详见 3.6 节。

#### 3.3.7 屏蔽词库
- 维护本域屏蔽词，支持增删改查。

#### 3.3.8 通知配置
- 编辑本域通知模板，配置事件模板映射与群发通知，启用/停用邮件渠道（使用平台配置的 SMTP）。

#### 3.3.9 域内日志
- **操作日志**：查看本域内的管理操作记录。
- **登录日志**：查看本域用户的登录记录。

### 3.4 工单系统

#### 3.4.1 工单类型与动态表单
- 系统预置类型模板：**问题、反馈、建议、投诉**，其中`反馈/建议`的前台独立入口见 3.7 节；`反馈/建议`作为工单底座上的专用类型，MVP 状态流固定为 3.7 节所述流程，不开放域管理员修改，其余类型可按域级配置调整默认状态流和基本字段。
- 域管理员可：
  - 基于预置模板创建新类型，或从零新建。
  - 使用**低代码表单设计器**添加动态字段（评估采用 Formily 或 react-jsonschema-form）。
- 工单模板用于内部客服/管理员快速创建工单，可预填标题、类型、动态字段默认值、默认优先级与附件说明，也可从现有工单配置快速复制。
- 内容模板用于客户端提交工单时手动应用，客户先选工单类型，再点击对应模板快速整理格式化内容；内容模板只预填内容，不自动提交。
- 动态字段属性：支持文本、多行文本、数字、单选、多选、日期、日期时间、附件、级联选择。可配置必填、客户可见、默认值、校验正则、排序权重。
- 系统固定字段：标题（必填）、描述（可选）、附件（可选）。
- 客户提交工单时不提供优先级选择，系统按工单类型和业务域规则默认赋予初始优先级；优先级仅由客服/管理员在处理过程中调整，紧急工单另行标记。
- 工单编号 `ticket_no` 由系统在创建工单时生成，格式为 `{business_domain.code}-{YYYYMMDD}-{seq}`；其中 `business_domain.code` 为稳定域短码，创建业务域时由系统按域名称生成候选值，管理员确认后锁定，后续域名变更不影响历史工单编号。
- 提交后动态字段数据以 JSON 存储，常用筛选字段生成虚拟列索引。

#### 3.4.2 状态流
- 每个工单类型绑定**唯一一个**状态流，最多定义 **10 个状态**。
- 已关闭、已撤回、已合并均为终态，不支持重开；如需继续处理，必须新建工单并可关联原单。
- 状态间支持分支转移（如“处理中”可到达“待补充”或“已解决”）。
- 系统自动检测**孤立状态**（无流入/流出、不可达、未形成闭环）并**警告域管理员**，但不强制删除，由业务自行决定。
- 所有状态变更由后端强校验，只允许配置图中的路径流转。

#### 3.4.3 工单处理流程
**客服端关键操作**：
- **工单队列**：按业务域、类型、状态、优先级、SLA 状态筛选；支持批量领取、批量指派、批量关闭和批量导出。批量操作支持部分成功，完成后展示成功/失败数量与失败原因摘要，不做全量回滚。
- **工单详情**：左侧对话流按 `ticket_reply.created_at` 时间线倒序展示（客户公开回复、客服内部备注、系统操作记录），右侧属性卡（状态、优先级、指派人、SLA 倒计时）。
- **快速操作**：回复客户、内部备注、转派、变更状态（二次确认）、请求补充信息、调整优先级、标记/取消紧急工单、合并到主单、插入快速回复结束语。
- **附件**：上传大小/类型限制，预览/下载留痕。
- **快捷回复模板**：客服关闭工单前可插入结束语或常用话术，模板由 `quick_reply_template` 统一管理，支持按业务域配置。
- **附件引用**：工单附件统一落 `file_attachment`，并通过 `attachment_ref` 关联到工单与回复记录。
- **并发控制**：工单实体包含 `version` 字段，提交时基于该字段做乐观锁校验；若版本冲突，提示“工单已被他人修改，请刷新”。
- **重复工单**：合并到主单后，被合并单进入已合并 / 已关闭终态，客户侧显示简短合并提示，主单继续处理；被合并单不生成新的满意度评价记录，待评价邀请仅在主单关闭后触发。
- **操作流留痕**：客户撤回、客服转派、改优先级、关闭、合并和系统自动流转统一写入 `ticket_history`，主体通过 `operator_subject_id` 与 `operator_actor_type` 记录，便于区分客户、客服和系统动作。

#### 3.4.4 SLA 与计时
- **双层策略**：域默认 SLA（兜底）+ 工单类型 SLA（覆盖默认）。
- **紧急工单权限**：仅客服/管理员可标记或调整紧急工单。
- **计时日历**：域管理员可配置工作日、节假日、自定义工作周（如周一至五 9:00-18:00，周六 9:00-12:00）。普通工单按其计算，**紧急工单忽略日历，按自然时间计时**（时限更短，具体值可配）。
- **暂停计时**：可指定某些状态（如“待客户补充”）暂停 SLA 时钟。
- **预警**：达到时限阈值时站内通知指派客服及域管理员，并支持额外配置接收角色/用户，列表显示红/黄标签。
- **违约动作**：域管理员可配置复选动作，包括预警通知、调高一级或调整到指定优先级、自动转给指定人员；自动转给指定人员采用协助/抄送模式，不改变当前主处理人。若当前工单已处于最高优先级，则跳过调级动作，仅执行其余违约动作。
- **SLA 字段**：SLA 计时与状态直接落在 `ticket` 主表，至少包含首响截止时间、解决截止时间、实际首响时间、实际解决时间、SLA 状态、累计暂停时长与当前暂停开始时间，供列表、详情和统计共用。
- **终止规则**：工单被合并、客户撤回或进入终态后，对应工单 SLA 立即停止，不再继续触发预警或升级。

#### 3.4.5 客户端我的工单
- 客户端提供“我的工单”入口，展示当前客户账号在当前业务域下的工单；若同一统一身份主体加入多个业务域，支持切换业务域查看。
- 列表支持分页、按工单状态筛选、按创建时间/最近更新时间筛选、按工单编号或标题关键词搜索。
- 状态筛选项由业务域状态流映射生成，默认提供“全部 / 待处理 / 处理中 / 待我补充 / 已解决 / 已关闭”等客户可读标签。
- 列表默认按最近更新时间倒序展示；客户可查看工单编号、标题、状态、最近回复时间和关联附件摘要。
- 详情页仅展示客户可见内容：状态流转、公开回复、客户可见附件、客服要求补充、评价入口/评价结果、关联咨询/转工单信息；内部备注、内部操作流和内部 SLA 细节不展示。
- 客户可在待分配 / 待处理状态下撤回工单；撤回后工单进入终态，不支持重开。
- 已解决或已关闭工单只允许查看，不允许客户追加回复。

#### 3.4.6 工单满意度评价
- 工单关闭后系统自动生成待评价记录，`satisfaction_rating.source_type` 固定为 `ticket`，评价对象为当前工单。
- 工单评价入口展示在工单详情页和通知中心待办中；客户也可从关闭后的工单详情页继续进入评价，避免错过邀请。
- 工单评价规则沿用满意度评价统一机制，评价维度为星级 + 标签 + 文字，`satisfaction_rating.source_type` 取 `ticket`，默认 3 小时窗口、到期前 30 分钟提醒一次、到期后自动失效；该窗口用于鼓励客户在工单刚关闭时即时反馈，避免评价滞后。
- 工单评价提交后，详情页展示评价结果；若评价记录已失效，则仅展示失效状态，不再允许提交。

### 3.5 在线咨询

#### 3.5.1 功能范围
为客户提供实时聊天入口，客服在工作台接待，支持转工单和历史查询。

#### 3.5.2 会话流程
1. 客户选择业务域后进入咨询窗口，先进入该域的咨询队列，系统生成排队编号并展示前方人数与粗略预估等待时间；队列策略默认严格 `FIFO`，不支持 `VIP` / 紧急插队；若当前无客服在线，客户仍可继续留言并看到知识引导和稍后回复提示。
2. 实时通信采用 `WebSocket`，会话创建、历史查询、附件上传和关闭操作通过 HTTP API 完成；`WebSocket` 握手采用短时一次性 `ws_ticket` 鉴权，客户端先通过 HTTP 换取 `ws_ticket` 再建立连接，`ws_ticket` 默认 60 秒内有效且单次使用，校验成功后立即作废；连接异常时前端自动重连，必要时回退到轮询获取最新消息。
3. 咨询客服状态分为 `在线 / 忙碌 / 离线`：客服上线后默认在线，当同时接待数量达到上限时自动切换为忙碌，接待数回落后自动回到在线，离线状态不再接入新会话。
4. 会话进入域未分配队列后，客服手动接入（MVP 阶段不做自动分配）。若当前无客服在线或客服已达同时接待上限，客户继续排队并保留留言，系统返回知识引导和稍后回复提示。
5. 客服同时接待上限默认 `5`，支持按业务域配置。
6. 实时消息双向传输，显示已读/未读状态；MVP 暂不展示“对方正在输入...” typing indicator。
7. 客服可使用**快捷回复**预设文本，或**引用知识库条目**发送给客户。
8. 客户与客服都可撤回自己发送的消息，限制在发送后 `2` 分钟内；撤回后双方均显示“消息已撤回”。
9. 同一咨询会话在多浏览器标签页同时打开时，仅保留一个活跃 `WebSocket` 连接；后打开标签页接管后旧连接自动关闭，避免重复收发消息与已读状态冲突。
10. 咨询评价规则沿用满意度评价统一机制，评价维度为星级 + 标签 + 文字，`satisfaction_rating.source_type` 取 `consultation_session`，默认 3 小时窗口、到期前 30 分钟提醒一次、到期后自动失效；该窗口与工单评价保持一致，便于统一体验与通知策略。
11. 活跃会话默认在最后一条消息后 `10` 分钟无新消息自动关闭，支持按业务域自定义。
12. **转接客服**：客服可将进行中的会话转接给指定客服或回退到队列；转接后原客服退出会话，新客服接管，客户侧显示“您的咨询已转接给 XXX”或“您的咨询已重新排队”，会话消息历史完整保留并写入转接记录。
13. **转工单**：客服在会话中点击“转为工单”，系统自动带入客户信息、会话摘要及附件，选择工单类型后提交，原会话关联新工单 ID。

#### 3.5.3 数据与安全
- 咨询消息持久化存储，支持按时间倒序查看历史会话。
- `consultation_message.type` 固定枚举为 `text` / `image` / `attachment` / `system` / `knowledge_reference`；其中 `system` 仅表示会话内系统提示，不承载平台级站内通知。
- 咨询消息撤回需落库记录撤回时间，撤回后不删除原消息，只在会话流中显示撤回占位。
- 咨询转接记录需落库，保留原客服、新客服、转接原因、转接时间与目标队列/目标客服，不删除原会话消息。
- 咨询转接后 `consultation_session.agent_staff_account_id` 更新为当前接手客服，历史接待人通过 `consultation_transfer_log` 追溯。
- 咨询消息的文件传输限制直接复用全局附件策略，不单独定义一套咨询专用限制。
- 咨询附件统一落 `file_attachment`，并通过 `attachment_ref` 关联到咨询会话或具体消息。
- 客服在线 / 忙碌 / 离线状态统一由 `staff_presence` 持久化，用于排队接入、当前接待数和心跳超时判断。
- 关键字屏蔽对咨询消息同样生效：后端静默检测，命中则拒绝发送，返回通用提示。
- 会话配置由业务域维度统一管理，至少包含通信方式、客服同时接待上限、活跃会话超时等参数。
- `WebSocket` 长连接不在产品层固定单机上限，部署侧需支持水平扩展与压测定容，具体承载量由实施阶段根据资源和消息频率评估。
- 客户端在工单关闭或咨询结束后，可在通知中心和详情页收到评价邀请；默认评价窗口为 3 小时，到期前 30 分钟提醒一次，超时自动标记为失效，不影响后续正常服务。
- 咨询会话状态含义如下：`queued` 表示排队中，`active` 表示进行中，`ended` 表示会话正常结束（客服手动结束或超时自动结束），`closed` 表示最终归档态（评价完成或评价窗口到期后进入）。

#### 3.5.4 客户端咨询历史
- 客户端提供“咨询历史”入口，展示当前客户账号的历史咨询会话；若同一统一身份主体加入多个业务域，支持切换业务域查看。
- 列表支持分页、按会话状态筛选、按开始时间/最后消息时间筛选，默认按最近消息时间倒序展示。
- 列表展示业务域、接待客服、会话状态、开始时间、最后消息时间、是否转工单及关联工单编号。
- 会话状态默认提供“进行中 / 已结束 / 已关闭”等客户可读标签。
- 详情页展示完整会话消息、图片/附件、接待客服、开始/结束时间、转工单结果；进行中的会话可直接返回实时咨询窗口继续接入，已结束会话仅可查看历史。

### 3.6 知识库

#### 3.6.1 知识库模型
- **按归属分为两级**：

| 级别 | 说明 | 可见范围 |
|------|------|----------|
| **公共知识库** | 平台级知识库，跨业务域共享 | 所有业务域内客服均可查看 |
| **业务域知识库** | 归属于某个业务域的知识库 | 默认仅本业务域内部可见 |

- 每个业务域可创建多个业务域知识库。
- **文档存储**：一期采用 Markdown 格式存储，前端支持 Markdown 编辑与预览。

#### 3.6.2 权限规则
- **公共知识库**：
  - 创建权限：仅 `platform_admin` 可创建。
  - 内容管理：`platform_admin` 可指定维护人员（跨域），维护人员具备增删改查权限。
  - 查看权限：所有业务域的客服均可搜索、查看、引用；已发布的公开条目支持未登录用户访问。
- **业务域知识库**：
  - 创建权限：业务域管理员可在本域内创建。
  - 内容管理：**具备权限的域内客服**（通过角色权限项控制）可对本域知识库及库内条目进行增删改查。
  - 默认可见范围：**仅知识库内可见**（即本域内部）。
  - 可扩展为对客户端公开（见 3.6.3）。

#### 3.6.3 可见级别（业务域知识库）
- 业务域知识库在创建后默认**仅域内可见**，后续可由域管理员调整：

| 级别 | 说明 |
|------|------|
| `域内可见` | 仅本业务域内授权客服可见（**默认值**） |
| `公开` | 对客户端用户开放搜索和查看；若业务域本身为公开域，则未登录用户也可访问 |

- 可见级别在知识库粒度配置。

#### 3.6.4 知识库类型

| 类型 | 说明 |
|------|------|
| `帮助文档` | 面向客户的产品使用、FAQ 等 |
| `内部文档` | 面向客服的流程规范、话术模板等 |
| `案例库` | 典型工单案例，支持分类检索 |
| `自定义` | 域管理员可创建自定义类型（预留） |

#### 3.6.5 库内引用
- 一篇知识条目可通过链接引用同库或**跨库**的其他条目（公共知识库可被所有域引用）。
- **对外引用权限继承规则**：若客服在工单回复或咨询中，将某篇文档**引用给客户**，则：
  - 该文档对**当前客户**临时可见。
  - 若该文档引用了其他知识库的条目，被引用条目**默认继承引用方的权限**，也对该客户临时可见。
  - 临时可见范围仅限当前被引用的 URL，不暴露知识库目录结构。
- 被引用条目删除或下线后，引用方文档中对应位置显示“引用内容已不可用”占位提示，不阻断引用方文档本身的展示。
  - 临时分享有效期平台可配置，默认 15 天，对应写入 `article_share_log.expires_at`。

#### 3.6.6 管理与审核
- 条目内容：Markdown 格式编辑，支持图片/附件；已发布且公开的条目可生成 VitePress 风格的帮助中心页面。
- `slug` 规则：`knowledge_base`、`knowledge_category`、`knowledge_article` 的 `slug` 由系统按名称自动生成，中文标题优先转拼音首字母，英文统一小写并用 `-` 连接；首次保存 / 发布前允许手工调整一次，同一知识库内必须唯一，冲突时自动追加 `-2`、`-3`；发布后 `slug` 锁定，旧地址可保留别名 / 重定向。
- 草稿自动保存：编辑器默认按固定间隔自动保存草稿，页面异常关闭后可恢复最近草稿。
- 版本管理：通过显式保存 / 发布生成新版本，可查看历史版本及回滚；草稿自动保存不生成历史版本。
- 并发编辑：知识条目采用条目级写锁，同一时间仅允许 1 人编辑同一 `knowledge_article`；保存时仍基于 `knowledge_article.version` 做最终校验，锁通过心跳续租并在超时后自动回收，MVP 不做三方合并 / diff。
- 审核流：草稿 → 待审核 → 已发布 / 已驳回。客服可提交草稿，域管理员审核发布。
- 搜索与引用：客服通过侧边栏搜索本域及公共知识库已发布条目，支持一键引用到工单回复或咨询消息；搜索由全文检索引擎承载，详见 3.6.10。
- 知识附件统一落 `file_attachment`，并通过 `attachment_ref` 关联到知识条目。

#### 3.6.7 知识迁移
- 支持同业务域内不同知识库之间的知识条目迁移和整库迁移，具备知识库管理权限的域内用户可发起。
- 支持跨业务域迁移，仅 `platform_admin` 可发起。
- 迁移后，原作者、版本历史、审核状态和审计链路保留，条目归属目标知识库，目标库的可见规则和审核规则生效。
- 源域默认执行真正迁移，同时允许选择“复制迁移”模式。
- 迁移完成后，需要通知目标域知识管理员或域管理员。
- 迁移后，已分享给客户且仍在有效期内的链接继续可访问；条目下线或删除后立即失效。

#### 3.6.8 与工单的联动
- 工单关闭时，客服可点击“生成知识草稿”，系统预填摘要进入知识库待审箱。

#### 3.6.9 帮助中心与公开页面
- 已发布且设为公开的知识库条目可生成帮助中心公开页，页面风格参考 VitePress，支持独立访问与站内阅读。
- 公开页使用友好 URL，URL 由知识库、分类和条目标识组合生成；页面需输出 `title`、`description`、`canonical` 和 `sitemap` 所需信息。
- 帮助中心只读取已发布版本，不读取草稿或待审内容；公共知识与公开业务域知识共用同一套发布内容。
- 帮助中心采用静态生成 + 发布触发重建方案，构建工具链采用 VitePress：知识发布后由构建流水线触发重建并发布静态产物；正文页静态承载，搜索、统计和推荐等辅助能力可通过客户端接口动态补充。

#### 3.6.10 全文搜索
- 知识库全文搜索使用 Elasticsearch / OpenSearch，支持中文分词。
- 索引字段至少包含标题、摘要、正文、标签、分类名与知识库名，结果按相关度排序，支持按知识库、分类、状态、类型和可见级别筛选。
- 客户端搜索仅检索当前可见范围内的内容：当前业务域公开知识与公共知识库；默认不跨所有公开业务域全局混搜。
- 客服侧搜索结果优先展示已发布且可见的条目，支持一键引用到工单回复或咨询消息。

#### 3.6.11 分类层级
- `knowledge_category` 最多支持 3 层嵌套，超出层级不允许保存。
- 前端侧边栏和帮助中心目录按 3 层以内结构渲染，避免目录过深影响浏览与定位。

#### 3.6.12 并发编辑与草稿恢复
- 知识条目编辑时默认保存草稿，草稿内容写入 `knowledge_article_edit_session`，页面异常关闭后可恢复最近一次草稿。
- 知识条目采用条目级写锁，同一时间仅允许 1 人编辑同一 `knowledge_article`；进入编辑页先申请锁，成功后方可编辑，其他用户仅可只读查看并显示当前编辑人。
- 锁通过心跳续租并在超时后自动释放；关闭浏览器时前端尝试立即释放，但异常退出由超时回收兜底。保存时仍校验 `knowledge_article.version`；若版本变化或锁失效，提示条目已被他人修改，请刷新后重试。MVP 不做三方合并 / diff。

#### 3.6.13 阅读统计
- 已发布的知识条目记录阅读量与最近阅读时间，阅读次数直接累加到条目 `view_count`，支持用于判断条目价值与优化帮助中心内容。
- 阅读事件可写入阅读日志作为明细，但统计结果仅用于管理端分析，不对客户端暴露。

### 3.7 反馈/建议
- 反馈/建议独立入口，实际复用工单底座承载，不与工单混合，但沿用 `ticket:*` 权限项，不单独定义 `feedback:*` 权限项。

#### 3.7.1 客户提交入口
- 客户端入口为“反馈/建议”与“我的反馈”，仅登录客户可提交，提交结果进入当前业务域。
- 提交表单默认包含：反馈分类（必填，绑定反馈专用类型）、标题（必填）、描述（必填）、附件（可选，复用全局附件策略）、联系方式（自动带出且只读）。
- 反馈分类默认预置为“问题反馈 / 功能建议 / 投诉 / 其他”，域管理员可按业务域调整名称与启停。

#### 3.7.2 客户列表与详情
- 客户侧“我的反馈”入口放在客户端服务记录中，支持分页查看当前客户账号提交的反馈。
- 列表支持按状态、分类、时间和关键词筛选，默认按最近更新时间倒序展示。
- 详情页展示反馈内容、附件、公开回复、当前状态、创建时间、处理时间线和内部处理进度。

#### 3.7.3 管理端处理流
- 管理端复用工单队列能力进行处理，支持查看、筛选、搜索、领取、指派、内部备注、公开回复、关闭和导出。
- 状态流固定为：待评估 → 处理中 → 已处理；客户侧统一展示“已收到 / 处理中 / 已完成”，不展示“已采纳 / 已拒绝”结论。
- 内部在已处理阶段记录 `result`（`adopted` / `rejected`），仅内部可见；客户详情仅展示最终处理结果文案，不直接暴露采纳判断。

#### 3.7.4 补充评论
- 客户在待评估或处理中阶段可对原反馈追加补充说明，追加内容以新的 `ticket_reply` 记录进入时间线，不覆盖原始提交。
- 已处理或已完成后不再允许追加评论；如需继续反馈，客户应新建一条反馈。

#### 3.7.5 SLA 规则
- 反馈/建议复用工单 SLA 引擎，默认可按业务域和反馈类型配置首响时限、处理时限及违约动作。
- 若某反馈类型未单独配置 SLA，则沿用业务域默认 SLA；SLA 违约后的预警、调优先级和协助 / 抄送规则与工单保持一致。

### 3.8 安全与屏蔽
- **关键字屏蔽**：域管理员可维护本域屏蔽词库。工单提交、回复、咨询消息、反馈/建议、知识草稿等所有用户输入文本，后端静默检测，命中则拒绝提交并返回“内容包含受限信息”提示，记录审计日志。
- **XSS / CSRF 防护**：工单、咨询、反馈、知识正文、模板变量等所有用户输入在入库前执行白名单净化，展示时执行输出编码；若敏感写操作或刷新流程采用 Cookie / 表单态会话，则必须携带 CSRF Token 并校验 `Origin` / `Referer`，Bearer API 仍按 `Authorization` 头鉴权。
- **静态加密**：仅对核心敏感数据采用静态加密或列级加密，密码仅做不可逆哈希；普通业务内容不强制做字段级加密。附件默认走对象存储直传，不经 API 服务器中转；本地存储降级模式下走 API multipart 上传，并受文件类型、大小、鉴权下载、备份与清理策略约束。
- **权限变更即时失效**：当管理员修改用户角色、业务域成员关系、账号启用状态、密码或绑定手机号/邮箱后，相关作用域内已签发的访问令牌立即失效，用户需重新登录或刷新会话后重新获取权限；刷新令牌同步失效。实现上可通过 Redis 中的 `auth_version` 比对或令牌黑名单完成。
- **敏感操作二次验证**：高影响操作仅需重新输入密码；前端通过 `POST /api/v1/auth/step-up` 获取短时 `step_up_token`，敏感接口通过 `X-UD-Step-Up-Token` 提交。验证通过后，在默认 `15` 分钟内同账号同会话的同类敏感操作默认放行，可配置。删除业务域、身份归并、彻底删除员工/客户、全量导出、全局安全策略修改等操作每次仍需重新验证密码，后端按单次令牌校验。
- **敏感告警**：登录失败、找回密码频繁、咨询发消息/建工单/附件上传高频、关键词命中、账号锁定及删除/归并/导出等高危操作触发安全告警，写入 `security_alert`；告警仅在管理端安全告警中心可见，全局告警仅供 `platform_admin` 查看，域级告警可由本域 `super_admin` 查看，不向客户或普通域管理员主动推送。
- **审计日志**：所有管理操作、状态变更、权限修改、知识审核等均写入 `audit_log`，包含操作主体、主体类型、对象、动作、结果、时间；客户提交内容触发的屏蔽命中也写入 `audit_log`。`audit_log.operator_subject_id` 关联统一身份主体，`operator_actor_type` 区分 `staff` / `customer` / `system`。
- **自助安全审计**：找回密码、修改密码、手机号/邮箱改绑、客户注销等自助安全操作也写入 `audit_log`，便于追踪异常操作与合规审计。

#### 3.8.1 安全告警中心
- 安全告警中心对 `platform_admin` 可见全局告警；业务域 `super_admin` 可查看和处理本域告警。列表可按严重级别、告警类型、业务域、状态、时间范围筛选，支持关键字搜索。
- 告警状态流固定为：`待处理 → 已确认 → 已处置 / 已忽略`。`已确认` 表示已认领并复核，`已处置` 表示已采取修复/阻断/处理动作，`已忽略` 表示误报或无需处理。
- 告警处置支持单条和批量操作；批量操作仅能作用于同一筛选结果。
- `已处置` 与 `已忽略` 必须填写备注，记录处理结论与后续动作；`已确认` 可填写备注但不强制。
- 告警处理后保留原始告警记录、处理人、处理时间与关联审计日志，不做硬删除。
- 数据审计字段：所有核心表含 `created_by`、`updated_by`、`created_at`、`updated_at`。

### 3.9 通知中心
#### 3.9.1 事件-模板-渠道映射
- 通知中心采用“事件 -> 模板 -> 渠道 -> 收件人”映射表驱动，默认映射按事件类别配置，客户偏好仅覆盖非强制渠道。
- 客户端普通通知采用站内信持久化 + 未读数轮询机制，默认每 30 秒查询一次未读数；进入通知中心时立即同步最新列表。
- 普通通知不与在线咨询 `WebSocket` 共用连接，咨询实时消息继续使用独立会话通道。
- **账号安全类**（登录异常、找回密码、密码重置、手机号/邮箱改绑）：
  - 收件人：对应账号本人
  - 默认渠道：站内信 + 邮件，短信仅在通道启用时用于高风险场景
  - 规则：不支持退订
- **工单进度类**（工单创建、分配、状态变更、SLA 预警、补充请求、关闭）：
  - 收件人：客户、当前处理人、协同人
  - 默认渠道：站内信默认开启，邮件按偏好，短信仅关键预警
  - 规则：非安全邮件可退订
- **咨询类**（排队、接入、新消息、结束、转工单）：
  - 收件人：客户、接待客服
  - 默认渠道：站内信默认开启，邮件按偏好
  - 规则：非安全邮件可退订
- **反馈类**（反馈创建、领取、状态变更、关闭）：
  - 收件人：客户、处理人
  - 默认渠道：站内信默认开启，邮件按偏好
  - 规则：非安全邮件可退订
- **知识/分享类**（引用分享、临时可见到期、帮助中心更新）：
  - 收件人：被分享客户或订阅者
  - 默认渠道：站内信默认开启，邮件按偏好
  - 规则：非安全邮件可退订
- **域公告/系统公告类**（域公告、平台公告、维护通知）：
  - 收件人：域内用户或指定范围用户
  - 默认渠道：站内信默认开启，邮件按偏好
  - 规则：非安全邮件可退订

#### 3.9.2 模板变量
- 模板变量按事件上下文注入，缺失变量渲染为空，不中断发送；运营后台预览时需提供示例值。
- **全局变量**：`domain_name`、`domain_code`、`portal_name`、`current_date`、`current_time`
- **账号变量**：`customer_name`、`staff_name`、`login_name`、`customer_phone_masked`、`customer_email_masked`
- **工单变量**：`ticket_id`、`ticket_no`、`ticket_title`、`ticket_type`、`ticket_status`、`ticket_priority`、`ticket_url`、`assignee_name`、`sla_first_response_at`、`sla_resolution_at`
- **咨询变量**：`session_id`、`queue_no`、`agent_name`、`session_url`、`wait_position`、`estimated_wait_minutes`
- **反馈变量**：`feedback_id`、`feedback_category`、`feedback_status`、`feedback_url`
- **知识变量**：`article_title`、`article_url`、`base_name`、`share_expire_at`
- **公告变量**：`announcement_title`、`announcement_url`、`announcement_scope`
- **安全变量**：`event_name`、`ip`、`device`、`location`、`reset_url`、`bind_url`

#### 3.9.3 邮件退订
- 非安全类邮件默认提供退订链接，退订链接指向通知偏好页并仅关闭该类别的邮件接收。
- 安全类邮件不提供退订链接，且始终保留至少站内信触达。
- 退订仅影响邮件渠道，不影响站内信、短信和必须安全通知。

#### 3.9.4 批量通知
- 域管理员可对本域用户群发通知，平台管理员可跨域群发。
- 群发范围可按角色、标签、账号状态、业务域成员关系筛选，群发结果写入通知日志并可追踪。
- 群发任务支持草稿、待发送、发送中、已完成、已失败、已取消等状态；支持定时发送，发送开始前可取消，进入发送中后不再允许编辑内容。
- 群发任务应记录创建者、目标范围、模板、渠道、发送状态与统计结果。

#### 3.9.5 免打扰
- 不支持静默时间段或免打扰窗口配置，通知按事件实时发送。
- 客户偏好仅控制非安全邮件和可选通知渠道，安全类通知不受静默策略影响。
- 发送失败时仍按统一重试策略执行，最终失败记录并告警。

- **渠道**：站内信（默认开启）、邮件（需配置 SMTP，支持测试发送）、短信（可选，启用后可用于找回密码与手机号改绑）。
- **持久化**：站内信消息持久化至 `inbox_message`，用于收件箱列表、已读/未读与跳转查看；邮件、短信和站内信的发送轨迹与重试结果写入 `notification_log`。
- **重试策略**：通知发送失败时自动重试 3 次，最终失败后记录并告警；`notification_log` 保留每次发送结果与最后错误信息。
- **通知偏好**：客户可按通知类别配置站内信、邮件、短信接收偏好；账户安全类通知不允许关闭，非安全类邮件默认可退订。

## 4. 非功能需求

- **安全**：全站 HTTPS，Argon2 密码哈希，JWT + Refresh Token 轮换，附件签名访问，上传病毒扫描预留接口。
- **部署架构**：当前采用单机部署，应用保持模块化单体，服务边界、配置边界和数据边界保持清晰，预留后续拆分为独立服务的能力。
- **环境规划**：开发、测试、预发布、生产均部署在内部服务器上，分别使用独立配置、独立数据库和独立访问入口，环境之间互不干扰。
- **最低硬件推荐**：单机生产建议至少 `4 vCPU / 16 GB RAM / 200 GB SSD`，推荐 `8 vCPU / 32 GB RAM / 500 GB SSD`；资源预留需覆盖 Spring Boot、MySQL、Redis、Elasticsearch 与操作系统缓存。若资源低于最低推荐配置，仅允许开发 / 演示环境降级为 MySQL 全文检索，生产默认仍使用 Elasticsearch / OpenSearch。
- **CI/CD**：主干合并后自动构建、执行测试、生成版本化产物并部署到测试环境；预发布与生产通过审批后发布，所有发布产物支持回滚。
- **数据库迁移**：数据库版本升级统一使用 `Flyway`，仅允许前向迁移；发布前需完成备份校验，迁移脚本需与应用版本一并纳入流水线。
- **灰度发布**：新功能通过功能开关控制，可按业务域、角色或白名单灰度启停；灰度期间需保留关闭开关与快速回滚能力。
- **终端适配**：客户端与员工端均采用响应式布局，需兼顾手机、平板和桌面端；核心业务流程在常见窄屏下可正常使用，不能出现关键操作不可达或布局崩坏。
- **浏览器兼容性**：支持 Chrome / Edge / Firefox 最新两个主版本，以及 Safari 最新两个主版本；不支持 IE。若存在企业专用浏览器或旧内核环境，需在项目启动时单独确认兼容矩阵。
- **国际化**：MVP 仅支持简体中文；所有面向用户的文案、错误提示与状态提示统一通过文案资源管理，保留后续扩展多语言的结构。
- **无障碍**：本期不单独纳入 a11y / WCAG 验收要求。
- **数据治理**：工单、咨询消息、审计日志需有明确的保留、归档、导出与隐私政策披露规则；历史数据可只读查询，导出结果需脱敏。
- **性能**：工单列表常用查询 P95 < 1s；创建/状态流转/咨询消息 < 500ms；咨询消息送达延迟 < 200ms；首版容量通过压测定容，不在 PRD 中写死单机绝对并发数。
- **可用性**：P0 按单机部署验证健康检查、备份脚本、恢复手册和一次恢复流程；月度 99.9%、数据库主从、附件增量备份自动化与季度恢复演练作为生产上线 Gate。RTO 4 小时、RPO 24 小时作为恢复目标记录。
- **可观测性**：结构日志（JSON），全链路 `request_id`，监控错误率、延迟、SLA 达成率、数据库 / Redis 状态、备份结果与队列积压，并在管理端安全/运维告警中心集中展示。
- **技术栈**：React + Spring Boot + MySQL 8.0+ + Redis。

## 6. API 设计约定

- **错误码**：`code` 采用按模块预留号段的数字编码，建议统一为 5 位或 6 位；例如 `10000-10999` 认证/账号、`11000-11999` 工单、`12000-12999` 咨询、`13000-13999` 知识、`14000-14999` 通知、`15000-15999` 业务域/客户关系、`16000-16999` 权限/IAM、`17000-17999` 附件。P0 联调以 `doc/P0接口契约表.md` 头部号段声明为准。
- **版本**：路径 `/api/v1/`，不兼容变更提至 v2。
- **统一响应**：`{ "code": 0, "message": "success", "data": {...}, "request_id": "uuid" }`
- **分页**：参数 `page`，`page_size`（最大100），返回 `total`, `list`。
- **鉴权**：`Authorization: Bearer <token>`。Token 包含统一身份主体、账号类型、当前切换的业务域上下文、权限项清单及 `auth_version`；当角色、业务域成员关系或账号状态变化时，相关 `auth_version` 递增，旧 Token 立即失效。
- **登录**：登录请求需携带门户标识，用于区分客户端与员工端，并决定默认进入的门户。
- **健康检查**：`GET /api/v1/health` 用于存活检查，`GET /api/v1/readiness` 用于就绪检查；readiness 至少校验数据库、Redis 与当前 Flyway 版本。
- **关键路由示例**：
  - 工单：`POST /api/v1/domains/{domain_id}/tickets`
  - 咨询：`POST /api/v1/domains/{domain_id}/consultation/sessions`
  - 知识：`GET /api/v1/knowledge/articles?domain_id=...`
  - 管理配置：`/api/v1/admin/domains/{domain_id}/...`
  - 角色权限：`GET /api/v1/admin/domains/{domain_id}/roles` , `POST .../roles`

## 7. MVP 验收标准

1. 至少创建 3 个业务域，三种可见策略均可配置，三种注册策略（`open` / `invitation_only` / `admin_only`）均可配置且生效，用户与客户在不同业务域间的权限隔离正常。
2. 客户端和员工端分别登录，同一统一身份主体在已授权时可免重复登录切换门户，无权限时展示提示页。
3. 业务域直链访问可按目标域自动跳转，可见策略与注册策略组合生效，包括“公开 + 仅邀请注册”等场景的访问与注册行为符合预期。
4a. **[P0]** 客户入域支持手动添加、自注册、邀请码三种方式，待激活/激活/禁用状态流转正确。
4b. **[P1]** 在 4a 基础上支持云之家单点登录自动注册/绑定与身份归并，SSO 不可用时保留账号密码登录。
5a. **[P0]** 每个域可自定义工单类型，客户提单 → 客服领取/回复/关闭/转派 → 客户查单流程可正常闭环。
5b. **[P1]** 在 5a 基础上增加咨询接待、咨询转工单流程，与工单处理闭环联动。
6. **[P1]** 客户端可完成“提交工单 → 在线咨询 → 转工单 → 查看公开知识库”全流程；公开知识在启用域下可搜索与查看属于 P2。
7. 管理端实现公共库审核、引用、临时分享，公开条目支持未登录访问。
8. 管理端实现域库审核、引用、临时分享，域内可见规则生效。
9. 同业务域内知识条目迁移和整库迁移可用，具备知识库管理权限的域内用户可发起。
10. 跨业务域知识条目迁移和整库迁移可用，仅 `platform_admin` 可发起，迁移后审计链路、版本历史、作者信息保留。
11. 离职池覆盖到验收里，员工或域内用户进入离职池后，权限释放、历史记录保留，未关闭工单按既定策略回退到未分配队列。
12. 平台管理、域管理、角色与按钮级权限均可按配置生效，自定义角色仅能访问被授权的菜单和按钮。
13a. **[P0]** 关键字屏蔽、审计日志、SLA 基础计时与预警（首响/处理计时、临近/超时预警通知）均可追溯且可验证。
13b. **[P1]** SLA 暂停/恢复、违约动作完整实现（调优先级、协助/抄送）、通知自动重试与失败补偿均可追溯且可验证。
14. 交付 Swagger 文档、错误码表、部署手册。
15. 客户端可查看“我的工单”列表和详情，支持按状态、时间筛选及按工单编号/标题搜索，仅展示当前客户账号可见内容。
16. 客户端可查看“咨询历史”列表和详情，支持按状态、时间筛选，历史会话可回看消息、附件与关联工单信息。
17. 登录、注册、找回密码入口均具备验证码/滑块校验；单账号连续失败 5 次后锁定 10 分钟，找回密码单账号每日最多 3 次。
18. 咨询消息发送 `60` 条 / 分钟、附件上传 `20` 次 / 10 分钟且单文件 `200M`；高频发消息、频繁建工单、附件高频上传、关键词命中等场景会进入管理端安全告警中心。
19. 高影响敏感操作仅需密码 step-up，`POST /api/v1/auth/step-up` 可返回短时 `step_up_token`，敏感接口通过 `X-UD-Step-Up-Token` 校验；验证后 `15` 分钟内同类敏感操作默认放行，删除业务域、身份归并、彻底删除、全量导出和全局安全策略修改按单次令牌校验。
20. 客户端与员工端在手机、平板、桌面端下核心流程均可正常使用，响应式布局不出现关键内容溢出、遮挡或不可操作区域。
21. 全部面向用户的文案默认仅提供简体中文展示，并通过统一文案资源管理。
22. 工单、咨询消息、审计日志保留期限可配置，默认保留 `3` 年；超过热数据窗口后自动归档，归档数据可按工单号、客户、时间和业务域只读查询。
23. 管理端支持工单、客户和审计日志基础导出，导出结果脱敏、异步生成、文件有效期受控，并留下审计记录。
24. 客户端提供隐私政策/个人信息处理规则页面，注册或首次使用时可确认同意，并可查看版本与生效时间。
25. 单机部署下前端、后端、数据库和 Redis 可在内部服务器上独立部署与重启，模块边界保持清晰，后续可平滑拆分服务。
26. 开发、测试、预发布和生产环境均在内部服务器上独立运行，配置、数据库和访问入口互不干扰。
27. Token 策略默认值可通过系统设置调整，Access Token 默认 `30` 分钟、Refresh Token 默认 `7` 天、最大并发会话数默认 `5`、默认启用 Refresh Token 轮换且超限时踢除最旧会话。
28. Flyway 迁移脚本可随版本升级自动执行，升级前后 schema 与应用版本保持一致，迁移前具备备份校验。
29. 主干合并后 CI/CD 可自动构建、测试并部署测试环境，预发布与生产通过审批后发布，且支持回滚到上一版本。
30. `GET /api/v1/health` 与 `GET /api/v1/readiness` 可用，readiness 能反映数据库、Redis 与 Flyway 状态。
31. 功能开关可按业务域、角色或白名单灰度启停，预发布验证通过后可逐步放量。
32. 工单与咨询满意度评价均复用 `satisfaction_rating`，通过 `source_type=ticket` 与 `consultation_session` 区分来源；评价维度为星级 + 标签 + 文字，工单与咨询统一为 3 小时评价窗口、到期前 30 分钟提醒一次，到期后自动失效。
33. 重复工单可合并到主单，被合并单进入已合并 / 已关闭终态，客户侧可见合并提示，且被合并单 SLA 立即停止。
34. 客服在关闭工单前可插入快速回复结束语，平台默认模板可用，域管理员可按业务域配置模板。
35. 工单模板与内容模板分层生效：内部模板可用于快速创建工单，客户端内容模板按工单类型手动应用，不自动带出。
36. 域级优先级可增减并自定义显示文案，SLA 违约动作配置界面可用；P0 阶段运行时验证预警通知 + 自动调优先级，协助 / 抄送运行时执行归 P1（#13b）。
37. 工单队列支持批量领取、批量指派、批量关闭和批量导出。
38. 附件默认白名单覆盖常见图片、文档、压缩包格式，域管理员可继续增补类型并配置大小限制。
39. 无客服在线或客服达到上限时，咨询客户可看到排队编号、前方人数和粗略预估等待时间，并保持留言与知识引导入口可用。
40. 咨询客服状态支持 `在线 / 忙碌 / 离线` 切换，达到同时接待上限时自动切换为忙碌，空闲后自动回到在线。
41. `consultation_message.type` 固定为 `text / image / attachment / system / knowledge_reference`，且 `system` 仅表示会话内系统提示。
42. 客户与客服均可在发送后 `2` 分钟内撤回自己发送的消息，撤回后会话流显示“消息已撤回”。
43. 咨询文件传输限制直接复用全局附件策略，不单独配置咨询专用限制。
44. 客服关闭会话后可邀请客户评价，评价维度为星级 + 标签 + 文字，星级必填；默认评价窗口 `3` 小时，到期前 `30` 分钟提醒一次，超时自动失效。
45. 知识库正文采用 Markdown 格式存储与编辑，已发布内容可生成 VitePress 风格的帮助中心公开页，支持友好 URL、页面 `meta` 与 `sitemap`。
46. 知识库全文搜索由 Elasticsearch / OpenSearch 承担，支持中文分词、相关度排序与按知识库/分类/状态/类型筛选，客服可直接搜索并引用已发布条目。
47. `knowledge_category` 最多支持 3 层分类嵌套，超出层级不允许保存，帮助中心目录按 3 层以内结构展示。
48. 知识条目编辑采用条目级写锁，同一时间仅 1 人可编辑；编辑中草稿自动保存到 `knowledge_article_edit_session`，页面异常关闭后可恢复最近草稿；保存时基于 `knowledge_article.version` 校验，版本变化或锁失效时提示刷新重试，MVP 不做三方合并。
49. 已发布知识条目可记录阅读量与最近阅读时间，阅读量采用累加方式写入 `view_count`，管理端可查看条目阅读统计用于判断知识价值与优化帮助中心内容，客户端不展示该统计。
50. 反馈/建议拥有独立前台入口“反馈/建议”和“我的反馈”，提交表单至少包含分类、标题、描述和附件，客户可查看自己的反馈列表与详情。
51. 管理端可对反馈进行筛选、搜索、领取、指派、内部备注、公开回复、关闭和导出，状态流与处理结果可追溯。
52. 客户在待评估或处理中阶段可对反馈追加补充说明，追加内容进入同一反馈时间线，已处理后不允许继续追加。
53. 反馈/建议复用工单 SLA 引擎，支持按业务域和反馈类型配置首响/处理时限及违约动作。
54. 反馈/建议内部处理结论仅在管理端显示，客户侧仅展示已收到/处理中/已完成，不直接展示 adopted/rejected 结论。
55. 通知中心具备事件-模板-渠道默认映射表，账号安全、工单、咨询、反馈、知识和公告类事件的默认收件人和渠道均可追踪。
56. 非安全类邮件默认提供退订链接，点击后仅关闭该类别邮件接收，安全类邮件不提供退订。
57. 通知中心不支持静默时间段或免打扰窗口配置，通知按事件实时发送，失败后仍执行统一重试策略。
58. 通知模板支持全局、账号、工单、咨询、反馈、知识、公告和安全等分组变量，缺失变量不影响发送。
59. 域管理员可对本域用户群发通知，平台管理员可跨域群发，且群发任务可追踪创建者、范围、模板与发送结果。
60. 每个业务域最多可创建 20 个自定义角色，超过上限时系统禁止继续创建。
61. 需要数据范围控制的能力均通过权限项后缀表达（如 `self` / `domain_all`），角色编辑页不再单独提供数据范围配置面板。
62. 用户角色变更仅支持单个用户逐一调整，不支持批量角色变更；权限项由平台统一预置，域管理员不可自行新增权限项代码。
63. 应用运行日志默认保留 `180` 天，安全告警日志默认保留 `365` 天，且日志文件统一落在固定目录。
64. Redis 仅用于会话、`auth_version`、验证码、限流、热点配置、未读数和临时队列状态，不作为核心业务主存储。
65. P0 完成单机部署下健康检查、备份脚本、恢复手册与一次恢复流程验证；RTO 默认 `4` 小时、RPO 默认 `24` 小时作为恢复目标记录。月度 `99.9%`、数据库主从和季度恢复演练作为生产上线 Gate。
66. CDN 本期不启用，静态资源与附件可通过源站 / 对象存储直链访问。
67. 监控告警中心可查看接口错误率、延迟、数据库 / Redis 状态、备份结果、队列积压与关键安全事件。
68. 从业务域复制和提炼模板功能可生成并复用 `domain_template`，模板快照可再次用于新建业务域。
69. 工单、咨询和知识条目附件统一通过 `file_attachment` + `attachment_ref` 管理，附件策略与业务对象解耦。
70. 登录成功/失败记录统一落 `login_log`，可按账号、门户、IP、时间和结果筛选。
71. 平台级系统参数与域级零散参数分别可通过 `system_config` 与 `domain_config` 维护。
72. 工单与咨询可共用 `quick_reply_template`，客服关闭工单或会话时可插入快捷回复。
73. 咨询客服在线 / 忙碌 / 离线状态统一由 `staff_presence` 记录，且可用于接待数和心跳判断。
74. 客户端与员工端在 Chrome / Edge / Firefox 最新两个主版本以及 Safari 最新两个主版本下均可正常使用，不支持 IE。
75. 员工账号 CSV 批量导入支持 UTF-8 表头模板、必填字段校验、部分成功与错误报告下载，导入任务可追踪。
76. 工单、咨询、反馈和知识正文的用户输入均有 XSS / CSRF 防护策略，敏感写操作的 Cookie / 表单态请求必须通过 CSRF Token 与来源校验。
77. 员工端与平台后台支持 IP 白名单控制，命中白名单外 IP 时拒绝登录并写入日志 / 告警，客户端不受影响。
78. `WebSocket` 握手可通过短时一次性 `ws_ticket` 完成鉴权，`ws_ticket` 单次有效且超时即失效。
79. 知识库 `slug` 由系统自动生成，首次保存 / 发布前可手工调整一次，同一知识库内保持唯一且发布后锁定。
80. 咨询队列默认严格 `FIFO`，不支持 `VIP` / 紧急插队。
81. `ticket_history` 统一记录客户、客服和系统动作，`operator_subject_id` 与 `operator_actor_type` 可区分操作主体。
82. `platform_admin` 手动授予 / 回收业务域 `super_admin` 时必须通过敏感操作二次验证并写入审计日志；平台管理员新建业务域后系统自动将创建者设为当前域 `super_admin`，该自动授权不视为违规自授。
83. 工单关闭后自动生成待评价记录，客户可从工单详情页和通知中心进入评价，默认 3 小时评价窗口、提前 30 分钟提醒、到期自动失效。
84. 安全告警中心支持按严重级别、类型、业务域、状态和时间筛选，支持单条与批量处置，`已处置` / `已忽略` 必须填写备注且处置过程可追踪。
85. 业务域软删除后，`super_admin` / `platform_admin` 仍可通过历史查询与导出入口只读访问该域历史工单、咨询、知识与审计数据，普通域用户不可见。
86. 数据归档 / 导出任务由归档中心或列表页入口触发后生成 `data_archive_job` / `data_export_job`，可查看状态、进度、完成时间、失败原因与文件有效期。
87. API 错误码采用按模块预留号段的数字编码，形成统一错误码表，便于前后端联调与日志排查。
88. 同一咨询会话在多浏览器标签页同时打开时仅保留一个活跃 `WebSocket` 连接，后打开标签页接管后旧连接自动关闭，不产生重复消息或状态冲突。
89. `attachment_policy.max_total_size_mb` 按单个业务对象维度生效，工单、咨询会话和知识条目的附件总量分别累计。
90. 客户端知识搜索仅检索当前可见范围：当前业务域公开知识与公共知识库，不默认跨所有公开业务域全局混搜。
91. 平台工作台首页默认展示全平台运营摘要，包含业务域总数、在线客服总数、排队客户数、待处理工单数、安全告警数与归档/导出任务状态，并提供平台管理页入口。
92. 域工作台首页默认展示本域运营摘要，包含今日工单、待处理工单、在线客服数、排队客户数、SLA 达标率与本周反馈趋势，并提供工单、咨询、反馈和知识库入口。
93. 咨询会话支持转接给指定客服或回退到队列，转接后原客服退出、客户侧显示转接提示且消息历史完整保留。
94. 客户端普通通知通过站内信未读数轮询感知新消息，默认每 30 秒查询一次未读数；普通通知与在线咨询 `WebSocket` 通道分离。
95. 附件默认通过 MinIO 作为对象存储，客户端直传并支持分片上传与断点续传；若 Sprint 0 未确认 MinIO，则 P0 以本地文件存储模式交付，API 支持 multipart 上传、鉴权下载、附件落库、本地备份清理与后续迁移到对象存储，迁移后 `attachment_id` 与业务引用保持不变。
96. 单机生产环境最低硬件配置建议不低于 `4 vCPU / 16 GB RAM / 200 GB SSD`，推荐 `8 vCPU / 32 GB RAM / 500 GB SSD`；知识库搜索默认使用 Elasticsearch / OpenSearch，低于最低配置时仅允许开发 / 演示环境降级为 MySQL 全文检索。
97. 员工端与客户端的密码复杂度策略可按 `portal_type` 分别配置，默认强度与最小长度可不同，且系统设置中可独立调整。
98. `audit_log` 支持按业务域筛选；平台级操作可使用 `domain_id = NULL`，域内操作需写入对应 `domain_id`。
99. `ticket_reply` 按 `created_at` 排序展示时间线，支持工单详情页按追加时间查看公开回复、内部备注和系统记录。
100. 业务域至少保留 1 个 `domain_admin`，最后一个 `domain_admin` 不允许自行降级或移除，后端必须拒绝并提示需要先指定另一位域管理员。
101. 每个业务域始终至少保留 1 个激活的 `super_admin`，最后一个域级 `super_admin` 不允许被禁用、移除或降级，后端必须拒绝并提示需要先指定另一位业务域超级管理员。
102. 被合并工单不生成新的满意度评价记录，待评价邀请仅在主单关闭后触发。
103. SLA 违约时若当前工单已处于最高优先级，则调级动作自动跳过，仅执行预警、协助 / 抄送等其余违约动作。
104. 客户多个待评价邀请在通知中心按创建时间或邀请时间倒序独立展示，不做聚合展示。
105. 被引用知识条目删除或下线后，引用方文档中对应位置显示“引用内容已不可用”占位提示，但不阻断引用方文档本身的展示。
106. 帮助中心公开页采用静态生成 + 发布触发重建方案，构建工具链采用 VitePress，知识发布后构建流水线自动触发重建并发布静态产物，正文页静态承载，搜索 / 统计 / 推荐可通过客户端接口动态补充。
107. 知识条目采用条目级写锁，同一时间仅允许 1 人编辑同一 `knowledge_article`；编辑内容自动保存到 `knowledge_article_edit_session`，锁可通过心跳续租并在超时后回收，页面异常关闭后可恢复最近草稿。
108. 咨询转接记录需落库到 `consultation_transfer_log`，并在转接后更新当前接手客服，历史接待人可通过转接日志追溯。
109. 咨询会话状态需明确区分 `queued` / `active` / `ended` / `closed`，其中 `ended` 为正常结束，`closed` 为最终归档态。
110. 批量领取、批量指派、批量关闭和批量导出支持部分成功，完成后需展示成功 / 失败数量及失败原因摘要，不做全量回滚。
111. 平台必须始终至少保留 1 个激活的 `platform_admin`；最后一个 `platform_admin` 不允许被禁用、删除、离职释放、降级或回收平台角色，后端必须拒绝并提示需要先指定另一位平台管理员。
