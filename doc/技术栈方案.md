# 技术栈方案（UnionDesk）

| 版本 | 日期 | 负责人 | 说明 |
|---|---|---|---|
| V1.0 | 2026-04-06 | 架构组 | 明确当前项目前后端、数据、日志与部署技术选型 |

---

## 1. 总体原则

1. 优先选择成熟、社区活跃、团队易维护的技术。
2. MVP 阶段采用模块化单体，避免过度拆分。
3. 所有业务数据按 `business_domain_id` 做域隔离。

---

## 2. 前端技术栈（UnionDeskWeb）

- 框架：React + TypeScript
- UI 组件库：Ant Design
- 工程化：Vite
- 路由：React Router
- 状态管理：Zustand（或 Redux Toolkit，按团队习惯二选一）
- 接口层：基于 OpenAPI 生成类型化 API Client

---

## 3. 后端技术栈（UnionDesk）

- 运行时：Java 21+（目标 Java 25 LTS，按团队可用版本就近落地）
- 框架：Spring Boot
- 鉴权：Spring Security + JWT（Access Token + Refresh Token）
- 持久层：MyBatis（或 MyBatis-Plus）
- 数据库迁移：Flyway
- 缓存：Redis
- 异步任务：消息队列（RabbitMQ，后续可按规模评估 Kafka）

---

## 4. 数据与存储

- 关系型数据库：MySQL 8.x（推荐 8.4 LTS）
- 对象存储：S3 协议（生产可接云存储，开发可用 MinIO）
- 数据库字符集：`utf8mb4`

---

## 5. 日志与可观测性

1. 默认方案：ELK（Elasticsearch + Logstash + Kibana）。
2. 回退方案：若未部署 ELK，使用 Spring 日志（Logback）输出本地滚动文件。
3. 建议日志分类：应用日志、业务日志、审计日志。
4. 建议统一透传 `request_id` 便于链路追踪。

---

## 6. 部署与运维

- 容器化：Docker
- 网关入口：Nginx / Ingress
- 传输安全：HTTPS
- 环境分层：`dev`、`test`、`staging`、`prod`

---

## 7. 当前确定项（本次确认）

- 前端统一采用 React + Ant Design。
- 数据库迁移工具固定为 Flyway。
- 日志采用“ELK 优先，本地文件回退”策略。
